This page guides you through creating and testing your first AI voice agent. In about 10 minutes, you will have a working appointment reminder agent that you can speak to in the browser. No telephony setup is required.

## Prerequisites

Make sure you have the following:

- An active Agora account and project
- Enabled Conversational AI for your Agora project
- An API key for a supported ASR provider
- An API key for a supported LLM provider
- An API key for a supported TTS provider

## Create an agent

1. Log into [Agora Console](https://console.agora.io) and open Convo AI Studio.
2. Select a project from your dashboard.
3. Click **Create Agent**.
4. Select the **Appointment Reminder** template.
5. Enter a name for your agent, for example, `My Appointment Bot`.
6. Click **Create**.

Studio creates the agent with a system prompt, greeting message, and default provider settings already configured.

## Configure your agent

1. Add ASR credentials

    1. Select **ASR** in the left panel or in the flow diagram.
    1. Select your ASR vendor from the dropdown.
    1. Under **Credential**, select **Add New Credential**.
    1. Enter a descriptive credential name.
    1. Enter the API key.
    1. Click **Save**.

    {/* IMAGE PLACEHOLDER: studio-quickstart-credentials --> */}

1. Add LLM credentials

    Select **LLM** in the left panel or in the flow diagram. Follow the same steps as for ASR to add a new credential for your LLM provider.

1. Add TTS credentials

    Select **TTS** in the left panel or in the flow diagram. Follow the same steps as for ASR to add a new credential for your TTS provider.

1. Review the system prompt (Optional)

    The system prompt defines how your agent behaves. Select **LLM** in the left panel and review the system instructions provided by the template. You can leave these as-is for the quickstart.

1. Review the greeting message (Optional)

    The greeting is the first thing your agent says when a call starts. Review the default greeting and optionally update it with a specific business name or agent name to make it more realistic during testing.

## Test the agent

1. Click **Start Test Session**.
2. Allow microphone access when prompted.
3. Speak to the agent as a caller would. Try confirming, rescheduling, and cancelling an appointment to test different conversation paths.

## Troubleshooting

If the test session does not work as expected, check the following:

- **No audio:** Confirm that your browser has microphone permission for Studio.
- **Agent not responding:** Verify that credentials are saved correctly for all three providers (ASR, LLM, TTS).
- **Unexpected behavior:** Review the system prompt for any variables that may need values filled in.
