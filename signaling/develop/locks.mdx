Critical resources can only be used by one process at a time. If a critical resource is shared between different processes, each process needs to be mutually exclusive to prevent interference with each other. Signaling provides a complete set of Lock solutions. By controlling different processes in the distributed system, you can solve the competition problem when users access shared resources.

<Vg k="SIG" /> provides the following APIs for managing locks:
<PlatformWrapper notAllowed="unity">
* [<code>{lock.set["android"]}</code>](../reference/api#locksetpropsag_platform): Create and configure a lock
* [<code>{lock.acquire["android"]}</code>](../reference/api#lockacquirepropsag_platform): Acquire the right to use a lock
* [<code>{lock.release["android"]}</code>](../reference/api#lockreleasepropsag_platform): Release a lock to make it available for others
* [<code>{lock.revoke["android"]}</code>](../reference/api#lockrevokepropsag_platform): Revoke an occupied lock to make it available for use
* [<code>{lock.get["android"]}</code>](../reference/api#lockgetpropsag_platform): Query lock information such as the total number of locks, lock name, lock user, and time to live
* [<code>{lock.remove["android"]}</code>](../reference/api#lockremovepropsag_platform): Remove a lock you no longer need
</PlatformWrapper>
