This page guides you through creating and testing your first AI voice agent using <Vpd k="STUDIO" />. In less than 10 minutes, you will have a working agent that you can speak to in the browser. No telephony setup is required.

## Prerequisites

Make sure you have the following:

- An active Agora account and project
- [Enabled Conversational AI](/conversational-ai/get-started/manage-agora-account) for your Agora project
- API keys for a supported ASR, LLM, and TTS provider. Refer to the list of [supported providers](supported-providers).

## Create an agent

To create your first agent:

1. Log into [Agora Console](https://console.agora.io) and open <Vpd k="STUDIO" />.
    ![](/images/conversational-ai/studio/create-agent.png)
1. In the left panel, select **Agents** then click **Create Agent**.
1. Select an Agora project from the dropdown. Agora uses this project for billing the usage for this agent.
    ![](/images/conversational-ai/studio/select-template.png)
1. Under **Choose a template**, select the **Appointment Reminder** template.
1. Enter a name for your agent, for example, `Appointment Reminder Agent`.
1. Click **Create**.

Studio creates the agent with a system prompt, greeting message, and default provider settings already configured.

## Configure your agent

### Add credentials

Follow these steps to add API keys for your ASR, LLM, and TTS providers to the agent.

- Add ASR credentials

    1. Select **ASR** from the left panel or the flow diagram.
    1. From the right panel, select your ASR **Vendor** from the dropdown.
    1. Under **Credential**, select **Add New Credential**.
    1. In the **Create Credential** dialog, enter a descriptive credential name.
    1. Enter the API key.
    1. Click **Save**.

    ![](/images/conversational-ai/studio/create-credential.png)

- Add LLM credentials

    1. Select **LLM** in the left panel or in the flow diagram. 
    1. From the right panel, select your LLM **Vendor** from the dropdown.
    1. Follow the same steps as for ASR to add a new credential for your LLM provider.

- Add TTS credentials

    1. Select **TTS** in the left panel or in the flow diagram. 
    1. From the right panel, select your TTS **Vendor** from the dropdown.
    1. Follow the same steps as for ASR to add a new credential for your TTS provider.

### Customize agent prompts (optional)

Templates come with pre-configured LLM system and greeting messages with variables that you can use for [telephony campaigns](/conversational-ai/studio/deploy/campaign). You can customize the agent by clicking on each component and updating the configuration as needed. This section guides you through customizing the system prompt and greeting message.

1. **System prompt** 

    The system prompt defines how your agent behaves. In the agent editor, select **LLM** in the left panel and review the default system instructions provided by the template in the panel on the right. You can leave these as-is for the quickstart.

1. **Greeting message** 

    The greeting is the first thing your agent says when a call starts. Review the default greeting and optionally update it with a specific business name or agent name to make it more realistic during testing.

## Test the agent

1. Click **Test Agent**.
2. Allow microphone access when prompted.
3. Speak to the agent as a caller would. Try confirming, rescheduling, and cancelling an appointment to test different conversation paths.

For guidance on what to test and how to troubleshoot issues, see [Test your agent](/conversational-ai/build/test-agent).
