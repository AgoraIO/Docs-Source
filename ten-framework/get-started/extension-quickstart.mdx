This page shows you how to create a Hello World extension for the <Vg k="AI_FRAMEWORK" />, step by-step. You can choose to build the extension in Python, Go, or C++ according to your coding preference.

## Prerequisites

Before starting, ensure that you understand the basics of the <Vg k="AI_FRAMEWORK" />. Following the [Quickstart](../get-started/quickstart) guide to build and run the <Vg k="AI_VOICE_AGENT" /> will get you up to speed quickly.

## Set up the project

Take the following steps to start the Docker containers:

1. **Compose up servers**

    Run the following command:

    ```bash
    docker compose up
    ```

    Once the command is entered, you should see output similar to this:

    ```bash
    ....
    Attaching to astra_agents_dev, astra_graph_designer, astra_playground
    astra_agents_dev      | >> run graph designer server
    astra_agents_dev      | cd agents && tman dev-server
    astra_agents_dev      | :-)  Starting server at http://0.0.0.0:49483
    astra_graph_designer  |   ▲ Next.js 14.2.4
    astra_graph_designer  |   - Local:        http://localhost:3000
    astra_graph_designer  |   - Network:      http://0.0.0.0:3000
    astra_graph_designer  | 
    astra_graph_designer  |  ✓ Starting...
    astra_playground      |   ▲ Next.js 14.2.4
    astra_playground      |   - Local:        http://localhost:3000
    astra_playground      |   - Network:      http://0.0.0.0:3000
    astra_playground      | 
    astra_playground      |  ✓ Starting...
    astra_graph_designer  |  ✓ Ready in 293ms
    astra_playground      |  ✓ Ready in 293ms
    ```

    At this point you have the following services running:

    - The backend server for the Graph Designer  
        `astra_agents_dev` at http://0.0.0.0:49483 

    - The UI of the Graph Designer  
        `astra_graph_designer` at http://localhost:3000

    - The UI of the <Vg k="AI_VOICE_AGENT" />  
        `astra_playground` at http://localhost:3001

1. **Enter the docker container**

    To work within the isolated environment, run the following command:

    ```bash
    docker exec -it astra_agents_dev bash
    ```

## Implementation

Take the following steps to implement the new extension:

1. **Create the extension**

    To create the `hello_world` extension, run the following command:

    <PlatformWrapper platform="python">
    ```bash
    cd agents/ten_packages/extension

    tman install extension default_extension_python \\
        --template-mode \\
        --template-data package_name=hello_world \\
        --template-data class_name_prefix=HelloWorld

    cd -
    ```
    </PlatformWrapper>

    <PlatformWrapper platform="go">

    ```bash
    cd agents/ten_packages/extension

    tman install extension default_extension_go \\
        --template-mode \\
        --template-data package_name=hello_world \\
        --template-data class_name_prefix=HelloWorld

    cd -
    ```
    </PlatformWrapper>

    <PlatformWrapper platform="cpp">
    ```bash
    cd agents/ten_packages/extension

    tman install extension default_extension_cpp \\
        --template-mode \\
        --template-data package_name=hello_world \\
        --template-data class_name_prefix=HelloWorld

    cd -
    ```
    </PlatformWrapper>

    After you run the command, you see a message similar to the following:

    ```bash
    ...
    Resolving packages...
    :-)  Installed successfully in xxx seconds
    ...
    ```

1. **Add API to the extension**

    Navigate into the `hello_world` directory and open `manifest.json`. Add the API objects with `data_in` and `cmd_out`. You use these within the Graph designer:

    ```json
    {
        "type": "extension",
        "name": "hello_world",
        "version": "0.4.1",
        "language": "python",
        "dependencies": [
            {
                "type": "system",
                "name": "rte_runtime_python",
                "version": "0.4.1"
            }
        ],
        "api": {
            "data_in": [
                {
                    "name": "text_data",
                    "property": {
                        "text": {
                            "type": "string"
                        },
                        "is_final": {
                            "type": "bool"
                        }
                    }
                }
            ],
            "cmd_out": [
                {
                    "name": "flush"
                }
            ]
        }
    }
    ```

1. **Build the extension**

    Switch back to the root of the project and build the extension using the following command.

    ```bash
    cd -

    make build
    ```

1. **Restart the server**

    You don’t need to restart the server when you first build the agent. However, after making subsequent updates, if refreshing the page doesn’t apply the changes, restart the server in Docker to ensure the updates take effect.

    ![](/images/ten-framework/restart-server.gif)

1. **Verify the extension**

    Open http://localhost:3001 in your browser. You see `hello_world` listed in the side bar. Drag it to the canvas, and connect it to the `text_data` input and `flash` output.

    You see green and red anchor points that indicate the possible node connection routes. 

    ![](/images/ten-framework/graph-designer-hello.gif)

    Congratulations! You have successfully created your first `hello_world` extension and integrated it into your agent using the Graph designer.

1. **Check the network requests**

    Open the Browser **DevTools**, navigate to the **Network** panel, and monitor the requests. You see the status codes returning as 200, indicating that the changes have been successfully processed.
