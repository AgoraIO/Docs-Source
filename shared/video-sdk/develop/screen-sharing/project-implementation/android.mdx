This section introduces how to implement <Vg k="SCR_SHR" /> in your project. The basic API call sequence is shown in the figure below:

![Screen Share Process](/images/video-sdk/screen-sharing-android-ios.svg)

There are two options to enable <Vg k="SCR_SHR" /> you can choose from according to the actual scenario:

- Call `startScreenCapture` before joining the channel, then call `joinChannel [2/2]` to join the channel and set `publishScreenCaptureVideo` to **true** to start <Vg k="SCR_SHR" />.

- Call `startScreenCapture` after joining the channel, then call `updateChannelMediaOptions` to update the channel media options and set `publishScreenCaptureVideo` to **true** to start <Vg k="SCR_SHR" />.

The flowcharts and steps in this article are based on the first scenario.

### Integrate <Vg k="SCR_SHR" /> plug-in

Screen sharing in the <Vg k="COMPANY" /> <Vg k="VSDK" /> is implemented through a plug-in, You can automatically integrate it through **Maven Central** or manually by importing an **aar** file to integrate the <Vg k="SCR_SHR" /> plug-in.

#### Automatic integration

When integrating the SDK through Maven Central, you can add dependencies by modifying the **dependencies** field in the `/Gradle Scripts/build.gradle(Module: <projectname>.app)` file:

    ``` java
    dependencies {
    // Please replace x.y.z in the following code with the specific SDK version number, you can get the latest version number from the release notes.
    def agora_sdk_version = "x.y.z"
    // 下述代码中包含 $，因此你必须使用 ""，不能使用 ''

    // Integrated solution 1
    implementation "io.agora.rtc:full-rtc-basic:${agora_sdk_version}"
    implementation "io.agora.rtc:full-screen-sharing:${agora_sdk_version}"
    implementation "io.agora.rtc:screen-capture:${agora_sdk_version}"

    // Integrated solution 2
    implementation "io.agora.rtc:full-sdk:${agora_sdk_version}"
    implementation "io.agora.rtc:full-screen-sharing:${agora_sdk_version}"
    ...
    }
    ```

#### Manual integration

1. Copy the `AgoraScreenShareExtension.aar` file from the SDK to the `/app/libs/` directory.

1. Add the following line to the **dependencies** node of the `/app/build.gradle` file to support importing files in the `aar` format:

    ``` java
    implementation fileTree(dir: "libs", include: ["*.jar","*.aar"])
    ```

1. Ensure that the `libagora_screen_capture_extension.so` file already exists in the `jniLibs` folder of your project. If the file does not exist, you need to manually copy it from the SDK folder.

1. Add the following code in the `/Gradle Scripts/build.gradle(Module: <projectname>.app` file to specify the location of the `JNI` library:

    ``` java
    android {
      // ...
      sourceSets {
        main {
          jniLibs.srcDirs = ['src/main/jniLibs']
        }
      }
    }
    ```

### Set up the audio scenario

Call the `setAudioScenario` method and set the audio scenario to `AUDIO_SCENARIO_GAME_STREAMING` (high-quality scenario) to improve the success rate of capturing system audio during <Vg k="SCR_SHR" />.

### Enable screen capture

Call `startScreenCapture` to start screen capture and set the parameters according to your application scenario:

    - `captureVideo`: Whether to capture system video during <Vg k="SCR_SHR" />.
    - `captureAudio`: Whether to capture system audio during <Vg k="SCR_SHR" />.
    - `captureSignalVolume`: The volume of the captured system audio.
    - `width`: Width (px) in video encoding resolution.
    - `height`: High (px) in video encoding resolution.
    - `frameRate`: Video encoding frame rate (fps).
    - `bitrate`: Video encoding bit rate (Kbps).
    - `contentHint`: Content type of <Vg k="SCR_SHR" /> video.

    ``` java
    // Set parameters for screen capture
    screenCaptureParameters.captureVideo = true;
    screenCaptureParameters.videoCaptureParameters.width = 720;
    screenCaptureParameters.videoCaptureParameters.height = (int) (720 * 1.0f / metrics.widthPixels * metrics.heightPixels);
    screenCaptureParameters.videoCaptureParameters.framerate = DEFAULT_SHARE_FRAME_RATE;
    screenCaptureParameters.captureAudio = screenAudio.isChecked();
    screenCaptureParameters.audioCaptureParameters.captureSignalVolume = screenAudioVolume.getProgress();
    engine.startScreenCapture(screenCaptureParameters);
    ```

### Publish a <Vg k="SCR_SHR" /> video stream in a channel

Call `joinChannel[2/2]` to join the channel and set the `options` parameter to publish the captured <Vg k="SCR_SHR" /> video stream as follows:

    ``` java
    ChannelMediaOptions options = new ChannelMediaOptions();
    options.clientRoleType = Constants.CLIENT_ROLE_BROADCASTER;
    options.autoSubscribeVideo = true;
    options.autoSubscribeAudio = true;
    // Avoid release of camera-captured video
    options.publishCameraTrack = false;
    // Avoid release of microphone-captured audio
    options.publishMicrophoneTrack = false;
    // Post screen-captured videos in the channel
    options.publishScreenCaptureVideo = true;
    // Post screen-captured audio in the channel
    options.publishScreenCaptureAudio = true;
    // Join the channel applying the channel media options set above
    int res = engine.joinChannel(accessToken, channelId, 0, options);
    ```

### Set up a <Vg k="SCR_SHR" /> scene (Optional)

Call the `setScreenCaptureScenario` method to set the <Vg k="SCR_SHR" /> scenario and choose the `screenScenario` that best fits your application from the following:

- `SCREEN_SCENARIO_DOCUMENT(1)`: Document scene
- `SCREEN_SCENARIO_GAMING(2)`: Game scene
- `SCREEN_SCENARIO_VIDEO(3)`: Video scene

``` java
engine.setScreenCaptureScenario(Constants.SCREEN_SCENARIO_VIDEO);
```

### Update <Vg k="SCR_SHR" /> (Optional)

If you want to update the <Vg k="SCR_SHR" /> parameters, such as the video encoding resolution, frame rate, or bit rate, call `updateScreenCaptureParameters` to modify the parameter values.

``` java
engine.updateScreenCaptureParameters(screenCaptureParameters);
```

### Stop <Vg k="SCR_SHR" />

Call `stopScreenCapture` to stop <Vg k="SCR_SHR" /> within the channel.

``` java
engine.stopScreenCapture();
```

### Limitations

Be aware of the following limitations:

- After turning on <Vg k="SCR_SHR" />, <Vg k="COMPANY" /> uses the resolution of the <Vg k="SCR_SHR" /> video stream as the billing standard. Please see [Pricing](../overview/pricing) for details. The default resolution is 1280 × 720, but you can adjust it according to your business needs.

- Due to <Vpl k="NAME" /> performance limitations, <Vg k="SCR_SHR" /> does not support <Vpl k="NAME" /> TV.

- Due to <Vpl k="NAME" /> system limitations, when using Huawei mobile phones for <Vg k="SCR_SHR" />, to avoid crashes, please do not adjust the video encoding resolution of the <Vg k="SCR_SHR" /> stream during the sharing process.

- Due to <Vpl k="NAME" /> system limitations, some Xiaomi phones do not support capturing system audio during <Vg k="SCR_SHR" />.

- On <Vpl k="NAME" /> 9 and later, to avoid system termination when the app is backed up, it is recommended to add the foreground service permission: `android.permission.FOREGROUND_SERVICE` to the `/app/Manifests/AndroidManifest.xml` file.

- Due to system limitations, screen capture is only available for <Vpl k="NAME" /> API level 21 and above, i.e., <Vpl k="NAME" /> 5 and above, otherwise the SDK will report error codes `ERR_SCREEN_CAPTURE_PERMISSION_DENIED(16)` and `ERR_ SCREEN_CAPTURE_SYSTEM_NOT_SUPPORTED(2)`.

- Due to system limitations, capturing system audio is only available for <Vpl k="NAME" /> API level 29 and above, i.e. <Vpl k="NAME" /> 10 and above, otherwise the SDK will report the error code `ERR_SCREEN_CAPTURE_SYSTEM_AUDIO_NOT_SUPPORTED(3)`.
