The <Vg k="FAST_SDK" /> includes default UI controls such as a toolbar, undo/redo, zoom, and page controls. You can configure these controls to display or hide as needed. For more advanced functionality, you can either customize the default controls or create and integrate your own custom controls into the whiteboard.

This guide explains how to use <Vg k="FAST_SDK" /> to customize controls and add custom UI components to the whiteboard.

## Understand the tech

The <Vg k="FAST_SDK" /> features a modular design that separates core functionality from the UI layer. This allows developers to either use the default UI controls or create custom ones. The default UI, including toolbars and page controls, is provided as configurable components in `FastboardUIConfig`. This layer manages rendering and interaction logic.

The following image shows some <Vg k="FAST" /> UI controls:

![Fastboard UI controls](/images/interactive-whiteboard/ui-widgets-fastboard.png)

- [Integrated](../get-started/get-started-uikit#add-dependencies) the <Vg k="FAST_SDK" /> into your project.
- Implemented the logic to [Join a Whiteboard Room](../get-started/get-started-uikit#join-a-whiteboard-room).

## Customize of <Vg k="FAST"/>'s controls

This section explains how to customize a default control or replace a default control with a custom control.

### Modify a default control

To show or hide controls, use the `FastboardUIConfig` object when creating a <Vg k="FAST"/> UI instance:

<Tabs groupId="language">
<TabItem value="javascript" label="Javascript" default>
<CodeBlock language="javascript" showLineNumbers>
{`// Create a UI instance
const ui = createUI(fastboard, container);

// Update the toolbar configuration
ui.update({
  config: {
    toolbar: {
      enable: true,
      placement: 'left',
      items: ["clicker", "selector", "pencil", "text", "shapes", "eraser", "clear"],
      apps: { enable: true },
    },
    redo_undo: { enable: true },
    zoom_control: { enable: true },
    page_control: { enable: true },
  },
});

// Hide the zoom control
ui.update({
  config: {
    zoom_control: { enable: false },
  },
});

// Set the toolbar to display only the pencil and eraser tools
const toolbar_items = ["pencil", "eraser"];
ui.update({
  config: {
    toolbar: { items: toolbar_items },
  },
});
`}
</CodeBlock>
</TabItem>

### Add a custom control

You can replace the default controls with custom UI controls. This section explains how to disable the default toolbar and add a custom toolbar.

#### Disable the default toolbar

To disable the default toolbar, set the `toolbar` property to `false`:

  // highlight-start
  toolbar: { enable: false },
  // highlight-end
  redo_undo: { enable: true },
  zoom_control: { enable: true },
  page_control: { enable: true },
}});`}

#### Use a custom toolbar

To add a custom control, follow these steps:  

1. Develop your custom UI component.  
2. Place the custom UI and the <Vg k="FAST"/> components in the same container.  
3. Pass the <Vg k="FAST"/> instance to the custom control to handle its logic.  

<Admonition>
- Call the APIs provided by the `FastboardApp` class to implement functions such as tool-switching, zooming, and page turning. To call these APIs, the room must have read and write permissions. For details, see [Development considerations](#development-considerations).
- <Vg k="COMPANY" /> provides the implementation code for the built-in Fastboard UI controls for your reference See [<Vg k="FAST"/> components](https://github.com/netless-io/fastboard/tree/main/packages/fastboard-ui/src/components/Toolbar/components).
</Admonition>

## Reference

### Development considerations

In custom UI control logic, when calling APIs in the <Vg k="FAST"/> App class (e.g., `setAppliance`), ensure you have read-and-write permissions. You can check this by monitoring the writable property of the <Vg k="FAST"/> App instance.

```javascript
let app = await createFastboard({ ...config });
console.log(app.writable); // { value: true, subscribe/reaction, set? }
```
#### `writable` property details

- **`value`**: The current permission state.
  - `true`: Interactive mode (read-and-write permissions).
  - `false`: Subscription mode (read-only permissions).
- **`subscribe` / `reaction`**: Add a listener to monitor permission changes.
  - `subscribe`: Immediately calls the listener with the current value and on every subsequent change. Returns a function to cancel the listener.
  - `reaction`: Calls the listener only when the value changes, without an initial callback.
- **`set`**: Updates the permission value. The change may not immediately reflect in `value`.

The following example demonstrates the difference between `subscribe` and `reaction`:

```javascript
// Add a listener with subscribe
let dispose = app.writable.subscribe((value) => {
  console.log("writable1:", value);
});
// Console output: writable1: true

// Add a listener with reaction
app.writable.reaction((value) => {
  console.log("writable2:", value);
});
// No initial console output

// Update the permission value
app.writable.set(false);
// Console output:
// writable1: false
// writable2: false

// Cancel the subscribe listener
dispose();

app.writable.set(true);
// Console output:
// writable2: true
```
