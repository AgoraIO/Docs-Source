<Vg k="COMPANY" /> <Vg k="VSDK" /> enables you to simultaneously join multiple channels. This capability allows you to receive and publish audio and video streams across multiple channels concurrently.

<PlatformWrapper platform="android, ios, macos, windows, unity, react-native, electron">

## Understand the tech

<Vg k="VSDK" />'s multi-channel functionality is based on two key components: 

* <code><Vpl k="RTC_CONNECTION" /></code>

    The <code><Vpl k="RTC_CONNECTION" /></code> object identifies a connection. It contains the following information:

    - Channel name
    - User ID of the local user

    You create multiple <code><Vpl k="RTC_CONNECTION" /></code> objects, each with a different channel name and user ID. Each <code><Vpl k="RTC_CONNECTION" /></code> instance can independently publish multiple audio streams and a single video stream.

* <code><Vpl k="RTC_ENGINE_EX" /></code> 

    The class contains methods tailored for interacting with a designated <code><Vpl k="RTC_CONNECTION" /></code> object.

    To join multiple channels, you call <code><Vpl k="RTC_JOIN_CHANNEL_EX" /></code> method in the <code><Vpl k="RTC_ENGINE_EX" /></code> class multiple times, using a different <code><Vpl k="RTC_CONNECTION" /></code> instance each time. 

When joining multiple channels:

- Ensure that the user ID for each <code><Vpl k="RTC_CONNECTION" /></code> object is unique and nonzero.
- Configure publishing and subscribing options for the <code><Vpl k="RTC_CONNECTION" /></code> object in <code><Vpl k="RTC_JOIN_CHANNEL_EX" /></code>.
<PlatformWrapper notAllowed="unity, react-native, electron">
- Pass the <code><Vpl k="RTC_EVENT_HANDLER_CLASS" /></code> object to the <code><Vpl k="RTC_EVENT_HANDLER_PARAMETER" /></code> parameter when calling the <code><Vpl k="RTC_JOIN_CHANNEL_EX" /></code> method to receive multiple channel-related event notifications.
</PlatformWrapper>
<PlatformWrapper platform="unity, react-native, electron">
- Use the `connection` parameter in each callback to identify the corresponding channel.
</PlatformWrapper>
