The <Vg k="RTEE_FC"/> extension collects quantitative data such as facial feature points, head rotation, and head translation. You can use this data to drive 3D facial stickers, headgear, pendant applications, or digital humans, adding more vivid expressions to virtual images.

This guide is intended for use-cases where the facial capture extensions is used independently to capture facial data, while a third-party rendering engine is used to animate a virtual human.

<Admonition type="info" title="Information"> 
To avoid collecting facial data through callbacks and building your own collection, encoding, and transmission framework, use the <Vg k="COMPANY"/> [MetaKit extension](../advanced-features/metakit) for facial capture.
</Admonition>

## Prerequisites

Ensure that you have:

- Implemented the [SDK quickstart](/video-calling/get-started/get-started-sdk) in your project.

<PlatformWrapper platform={['ios', 'macos']}>
- Integrated <Vg k="VSDK" /> version 4.3.0, including the face capture extension dynamic library `AgoraFaceCaptureExtension.xcframework`.
</PlatformWrapper>

<PlatformWrapper platform={['windows']}>
- Integrated <Vg k="VSDK" /> version 4.3.0, including the face capture extension dynamic library `libagora_face_capture_extension.dll`.
</PlatformWrapper>

<PlatformWrapper platform={['android']}>
- Integrated <Vg k="VSDK" /> version 4.3.0, including the face capture extension dynamic library `libagora_face_capture_extension.so`.
</PlatformWrapper>

- Obtained obtained the face capture authentication parameters `authentication_information` and `company_id` by contacting [<Vg k="COMPANY"/> technical support](mailto:extensions.marketplace@agora.io).
