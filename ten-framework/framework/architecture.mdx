The <Vg k="AI_FRAMEWORK" /> uses extensions developed in different programming languages as building blocks. These extensions are interconnected within a runnable server-side application, known as an Agent App, to execute more sophisticated operations. A Graph defines the relationships between the extensions and illustrates the flow of data. Sharing and downloading extensions is simplified through the <Vg k="AI_FRAMEWORK" /> Extension Store and Package Manager.

![](/images/ten-framework/framework-tech.png)

## Understand the tech

The following sections introduce the basic <Vg k="AI_FRAMEWORK" /> components.

### Extensions

An extension is the fundamental unit of composition within the <Vg k="AI_FRAMEWORK" />. You can create your own extensions and combine them with existing ones to build diverse scenarios and applications.

The <Vg k="AI_FRAMEWORK" /> emphasizes cross-language collaboration, enabling extensions written in different languages to work together seamlessly within the same application or service. For instance, if your application requires real-time communication (RTC) features and advanced AI capabilities, you might write the RTC-related extension in C++ for its performance advantages in processing audio and video data, while developing the AI extension in Python to leverage the platform's extensive libraries and frameworks for data analysis and machine learning tasks.


### Graphs

In the <Vg k="AI_FRAMEWORK" />, a Graph describes the data flow between extensions and orchestrates their interactions. For example, you can route the text output from a speech-to-text (STT) extension to a large language model (LLM) extension. Essentially, a Graph defines which extensions are involved and how data flows between them. You can customize this flow by directing outputs from one extension to another.

The <Vg k="AI_FRAMEWORK" /> supports the following types of data flow between extensions:

- Command
- Data
- Image Frame
- PCM Frame

By specifying the direction of flow in the Graph, you can enable mutual invocation and unidirectional data flow between extensions. This is especially useful for PCM Frame and Image Frame data types, as it simplifies audio and video processing.

### Agent app

An Agent app in the <Vg k="AI_FRAMEWORK" /> is a runnable server-side application that combines multiple extensions following Graph rules to accomplish more sophisticated operations. A <Vg k="AI_FRAMEWORK" /> Agent app is a robust application that executes complex operations by integrating multiple Extensions within a flexible framework defined by Graph rules. These Graph rules orchestrate the interplay between various extensions, enabling the app to perform sophisticated tasks that go beyond the capabilities of individual components.

By leveraging this architecture, a <Vg k="AI_FRAMEWORK" /> Agent app seamlessly manages and coordinates different functionalities, ensuring that each extension interacts harmoniously with others. This design allows you to create powerful and scalable applications capable of handling intricate workflows and data processing requirements.

### Extension Store

The <Vg k="AI_FRAMEWORK" /> Extension Store is a centralized platform designed to foster collaboration and innovation among developers. It provides a space where you can share your extensions, contribute to the community, showcase your work, and receive valuable feedback from peers, enhancing the overall quality and functionality of the <Vg k="AI_FRAMEWORK" /> ecosystem.

Beyond sharing your own extensions, you can access a wide array of extensions created by other developers. This extensive library makes it easier to find tools and functionalities that can be integrated into your projects, accelerating development and promoting best practices within the community. The <Vg k="AI_FRAMEWORK" /> Store thus serves as a valuable resource for both novice and experienced developers looking to expand their capabilities and leverage the collective expertise of the community.

### Package Manager

The <Vg k="AI_FRAMEWORK" /> Package Manager streamlines the process of managing extensions, making it easy to upload, share, download, and install them. It simplifies the workflow by allowing extensions to specify their dependencies on other extensions and the environment, ensuring that all necessary components are automatically managed and installed by the framework. This reduces the potential for errors and conflicts.

By automatically handling these dependencies, the <Vg k="AI_FRAMEWORK" /> Package Manager makes the installation and release of extensions both convenient and intuitive. This automation not only saves time but also enhances the user experience by ensuring seamless integration within the larger ecosystem. Consequently, this ease of use encourages the development and distribution of more robust and complex extensions, further enriching the <Vg k="AI_FRAMEWORK" /> framework.
