In under five minutes, you will have a working voice agent running locally. The guide covers installing the Agora CLI, cloning a starter project in your preferred language, and loading your Agora credentials.

To explore other ways to get started, see the related quickstarts:

- [CLI + SDK quickstart](/conversational-ai/get-started/quickstart-cli): Build the agent server yourself to understand how the pieces fit together.
- [SDK-only quickstart](/conversational-ai/get-started/quickstart-sdk): Skip the CLI and work directly with the SDK.

<Admonition type="info">
The starter projects use Agora-managed presets, so no vendor API keys are required to complete this quickstart. If you prefer to switch to your own vendor accounts, see the documentation for your chosen [ASR](/conversational-ai/models/asr/overview), [LLM](/conversational-ai/models/llm/overview), and [TTS](/conversational-ai/models/tts/overview) providers.
</Admonition>

## Prerequisites

- An Agora project with Conversational AI enabled. If you have not created one yet, follow along to create a new project with a single CLI command.
- One of the following runtimes:
  - **Node 20 LTS or later** for the TypeScript/Next.js starter
  - **Python 3.8 or later** and **Bun** for the Python starter
- **git** and a terminal.
- Roughly five minutes.

No Agora console configuration is required. The Agora CLI handles project creation, feature enablement, and credential retrieval.

## Set up your environment

This section walks you through installing the Agora CLI and signing in to your Agora account.

### Install the Agora CLI

Agora CLI is the recommended way to bootstrap a new Agora project. Use it to create projects, enable features, write credentials to `.env` files, and run a `doctor` diagnostic.

```bash
npm install -g agoraio-cli
```

Verify the installation and sign in with your Agora account:

```bash
agora --version
agora login
agora whoami
```

`agora login` opens a browser for SSO. On a headless machine, use `agora login --device` to complete the OAuth flow using a six-digit code.

<details>
<summary>Migrating from `agora-cli-preview`?</summary>

`agora-cli-preview` is deprecated. Uninstall it before installing the GA release:

```bash
npm uninstall -g agora-cli-preview
npm install -g agoraio-cli
```

Command names are unchanged in the GA release; configuration is re-read from `~/.agora-cli/` on first run.
</details>

### Create or select an Agora project

If you do not yet have a Conversational AI project, create one now:

```bash
agora project create my-agent-demo --feature rtc --feature convoai
```

If you already have a project, select it as the active project so subsequent commands use it:

```bash
agora project list
agora project use <project-id>
```

The CLI now has access to your App ID, App Certificate, and Conversational AI feature status. No console configuration is required.

## Clone and run a starter

Select a starter in your preferred language, load your Agora credentials, and run it locally.

Pick the tab matching your preferred runtime.

### Load your Agora credentials

Each starter ships a `.env` template at a slightly different path. The CLI writes your App ID and Certificate into the shell, and a single heredoc produces the final env file.

<Tabs groupId="language">
<TabItem value="js" label="TypeScript (Next.js)">

The Next.js starter expects `NEXT_PUBLIC_AGORA_APP_ID` and `NEXT_AGORA_APP_CERTIFICATE` in `.env.local`.

```bash
# Load Agora values into your shell from the active CLI project
eval "$(agora project env --shell)"

# Seed .env.local from the template, then overwrite with the real values
cp env.local.example .env.local
cat > .env.local <<EOF
NEXT_PUBLIC_AGORA_APP_ID=$AGORA_APP_ID
NEXT_AGORA_APP_CERTIFICATE=$AGORA_APP_CERTIFICATE
EOF
```

The remaining variables in `env.local.example` (e.g., `NEXT_PUBLIC_AGENT_UID`, `NEXT_LLM_*`, `NEXT_DEEPGRAM_API_KEY`, `NEXT_ELEVENLABS_API_KEY`) are **optional** — leave them blank to use Agora's managed presets.

</TabItem>

<TabItem value="python" label="Python (FastAPI + Next.js client)">

The Python starter expects `APP_ID` and `APP_CERTIFICATE` in `server-python/.env.local`.

```bash
# Load Agora values into your shell from the active CLI project
eval "$(agora project env --shell)"

# Seed server-python/.env.local from the template, then overwrite
cp server-python/.env.example server-python/.env.local
cat > server-python/.env.local <<EOF
APP_ID=$AGORA_APP_ID
APP_CERTIFICATE=$AGORA_APP_CERTIFICATE
PORT=8000
EOF
```

</TabItem>

<TabItem value="go" label="Go (coming soon)">

When the Go starter lands, this step will use `agora project env write .env --with-secrets` directly, matching the CLI + SDK variant.

</TabItem>
</Tabs>

<Admonition type="tip" title="Why not `agora project env write`?">
`agora project env write` is the fastest path when the target file uses Agora's default variable names (`AGORA_APP_ID`, `AGORA_APP_CERTIFICATE`). The Next.js and Python starters use framework-specific names (`NEXT_PUBLIC_AGORA_APP_ID`, `APP_ID`), so the `eval` + heredoc pattern above is more portable. If you own the starter repo, you can of course rename the vars and use `agora project env write` directly.
</Admonition>

### Run the app

<Tabs groupId="language">
<TabItem value="js" label="TypeScript (Next.js)">

```bash
pnpm dev
```

Open http://localhost:3000, click **Start conversation**, and speak. You should:

1. Hear a greeting within two seconds of clicking **Start**.
2. See live partial and final transcripts appear as you speak.
3. Hear the agent respond in a natural voice.

</TabItem>

<TabItem value="python" label="Python (FastAPI + Next.js client)">

```bash
bun run dev
```

This launches the FastAPI backend on http://localhost:8000 and the Next.js web client on http://localhost:3000. Open the web client, click **Start conversation**, and speak. You should:

1. Hear a greeting within two seconds.
2. See live transcripts in the UI.
3. Hear the agent respond.

FastAPI docs are live at http://localhost:8000/docs if you want to poke the backend directly.

</TabItem>

### Test and validate

Before you move on, confirm the three-point checklist below. A healthy run hits all three within the quoted time budgets.

| Check | How to verify | Budget |
|---|---|---|
| Agent joined the channel | The starter UI shows **"agent connected"** (Next.js) or the FastAPI log shows `agent started: agentId=...` (Python). | < 2 s |
| First audio arrives | You hear the greeting. | < 2 s after **Start** |
| Transcripts stream | Partial transcripts update word-by-word as you speak; final transcripts appear after each utterance. | < 500 ms partial-to-screen |

If any check fails, run the CLI doctor:

```bash
agora project doctor
```

`doctor` checks credential validity, feature enablement, and network reachability to Agora's edge — it catches most first-run issues before you start chasing code.

### Stop the agent

Close the browser tab, or `Ctrl+C` the dev server. The starter calls the `stop-conversation` endpoint on unload, which terminates the agent cleanly. The Convo AI engine will also idle-timeout after 30 seconds of silence.

To stop an orphaned agent by ID:

```bash
curl -X POST http://localhost:3000/api/stop-conversation \
  -H 'Content-Type: application/json' \
  -d '{"agentId":"<agent-id-from-logs>"}'
```

## Troubleshooting

First response when anything misbehaves: `agora project doctor`.

| Symptom | Likely cause | Fix |
|---------|--------------|-----|
| `agora: command not found` after `npm install -g agoraio-cli` | Global npm bin not on `$PATH`. | `npm prefix -g` → add `<prefix>/bin` to your shell rc. |
| `401 unauthorized` in the starter's browser console | `.env.local` / `.env.local` never picked up the CLI values. | Re-run the `eval` + heredoc block in Step 4; confirm the file contents with `cat`. |
| Greeting plays but no transcripts appear | `enable_rtm` is off, or your browser blocked the RTM WebSocket. | The starters set `enable_rtm: true` by default — check the browser console and your network policy for blocked `wss://` connections. |
| `pnpm: command not found` | pnpm is not installed. | `npm install -g pnpm` (TS tab) — the starter requires pnpm, not npm. |
| `bun: command not found` | Bun is not installed. | `curl -fsSL https://bun.sh/install \| bash` (Python tab) — the starter workspace requires Bun. |
| Agent greets itself in a loop | Microphone and speaker are on the same device without echo cancellation. | Use headphones, or set `parameters.enable_aec=true` in the starter's agent config. |
| `agora login` fails on CI or a locked-down box | Browser SSO cannot open. | Use `agora login --device` for a six-digit-code flow that works over SSH. |

If none of the above matches, open an issue on the starter repo with the output of `agora project doctor` attached — that output already contains the diagnostics the maintainers will ask for.
