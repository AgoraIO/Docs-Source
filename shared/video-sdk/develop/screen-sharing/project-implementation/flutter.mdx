This section introduces how to implement <Vg k="SCR_SHR" /> in your project.

### iOS platform

- Start <Vg k="SCR_SHR" />

    1. Set the parameters according to your application scenario and call `startScreenCapture` to start screen capture:

        - `captureVideo`: Whether to capture system video during <Vg k="SCR_SHR" />.
        - `captureAudio`: Whether to capture system audio during <Vg k="SCR_SHR" />.
        - `captureSignalVolume`: The volume of the captured system audio.
        - `dimensions`: Resolution of video encoding.
        - `frameRate`: Video encoding frame rate (fps).
        - `bitrate`: Video encoding bitrate (Kbps).
        - `contentHint`: Content type of <Vg k="SCR_SHR" /> video.

        ``` dart
        await rtcEngine.startScreenCapture(
            const ScreenCaptureParameters2(captureAudio: true, captureVideo: true));
        ```

    2. Combine this with the user's manual operation to make the <Vpl k="CLIENT" /> enable <Vg k="SCR_SHR" />. There are two methods for you to do this:

        - Method 1: Prompt the user to long press the **screen recording** button in the **Control Center** on iOS and choose to use the extension you created to start recording.
        - Method 2: Use Apple's new [RPSystemBroadcastPickerView](https://developer.apple.com/documentation/replaykit/rpsystembroadcastpickerview) in iOS 12.0 to make the **Enable Screen Sharing** button pop up on the <Vpl k="CLIENT" /> interface, prompting the user to click the button to start recording.

- Stop <Vg k="SCR_SHR" />

    Call `stopScreenCapture` to stop <Vg k="SCR_SHR" /> within the channel. The sample code is as follows:

        ``` dart
        await _engine.stopScreenCapture();
        ```

### Android platform

When enabling <Vg k="SCR_SHR" /> on Android, you only need to call the `startScreenCapture` method. You can refer to the `screen_sharing.dart` file in the <Vg k="COMPANY" /> <Vg k="VSDK" /> to implement <Vg k="SCR_SHR" />.

### macOS and Windows platforms

1. Get **screen ID** or **window ID**

    You can get **Display ID** or **Window ID** directly through `getScreenCaptureSources` provided in `agora_rtc_engine`. The sample code is as follows:

        ``` dart
        await rtcEngine.getScreenCaptureSources(
                thumbSize: thumbSize, iconSize: iconSize, includeScreen: true);
        ```

1. Share a specified screen or window

    The following sample code demonstrates how to share a specified screen on macOS or Windows using **Display ID**:

        ``` dart
        await rtcEngine.startScreenCaptureByDisplayId(
            displayId: sourceId!,
            regionRect: const Rectangle(x: 0, y: 0, width: 0, height: 0),
            captureParams: const ScreenCaptureParameters(
              captureMouseCursor: true,
              frameRate: 30,
            ));
        ```

    The following sample code demonstrates how to share a specified window on macOS or Windows using **Window ID**:

        ``` dart
        await rtcEngine.startScreenCaptureByWindowId(
          windowId: sourceId!,
          regionRect: const Rectangle(x: 0, y: 0, width: 0, height: 0),
          captureParams: const ScreenCaptureParameters(
            captureMouseCursor: true,
            frameRate: 30,
          ),
        );
        ```

1. Join the channel and publish a <Vg k="SCR_SHR" /> stream

    If you only need to publish the screen sharing stream, refer to the following code:

        ``` dart
        await _engine.joinChannelEx(
            token: '',
            connection: RtcConnection(
                channelId: _controller.text, localUid: shareShareUid),
            options: const ChannelMediaOptions(
              autoSubscribeVideo: true,
              autoSubscribeAudio: true,
              publishScreenTrack: true,
              publishSecondaryScreenTrack: true,
              publishCameraTrack: false,
              publishMicrophoneTrack: false,
              publishScreenCaptureAudio: true,
              publishScreenCaptureVideo: true,
              clientRoleType: ClientRoleType.clientRoleBroadcaster,
            ));
        ```

    To publish screen-shared streams and video streams captured by local cameras, add the following code to your project:

        ``` dart
        await _engine.joinChannelEx(
            token: '',
            connection:
                RtcConnection(channelId: _controller.text, localUid: localUid),
            options: const ChannelMediaOptions(
              publishCameraTrack: true,
              publishMicrophoneTrack: true,
              clientRoleType: ClientRoleType.clientRoleBroadcaster,
            ));


        await _engine.joinChannelEx(
            token: '',
            connection: RtcConnection(
                channelId: _controller.text, localUid: shareShareUid),
            options: const ChannelMediaOptions(
              autoSubscribeVideo: true,
              autoSubscribeAudio: true,
              publishScreenTrack: true,
              publishSecondaryScreenTrack: true,
              publishCameraTrack: false,
              publishMicrophoneTrack: false,
              publishScreenCaptureAudio: true,
              publishScreenCaptureVideo: true,
              clientRoleType: ClientRoleType.clientRoleBroadcaster,
            ));
        ```

### Limitations

Be aware of the following limitations:

- The video unit price for a screen-shared stream is based on the **video resolution** you set in `ScreenCaptureParameters`. If you do not pass dimensions in `ScreenCaptureParameters`, <Vg k="COMPANY" /> bills at the default resolution of 1920 x 1080 (2,073,600). See [Pricing](../overview/pricing) for details.

#### iOS platform

- Before starting <Vg k="SCR_SHR" />, call the `setAudioScenario` method and set the **audio scenario** to `audioScenarioGameStreaming` (high sound quality scenario) to improve the success rate of capturing system audio during <Vg k="SCR_SHR" />.
- Due to system limitations, <Vg k="SCR_SHR" /> is only supported on iOS 12.0 and above.
- This feature requires a high level of device performance. <Vg k="COMPANY" /> recommends that you use an iPhone X or above.

#### Android platform

- Before starting <Vg k="SCR_SHR" />, call the `setAudioScenario` method and set the **audio scenario** to `audioScenarioGameStreaming` (high sound quality scenario) to improve the success rate of capturing system audio during <Vg k="SCR_SHR" />.
