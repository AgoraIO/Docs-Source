## Implement cloud transcoding
This section walks you through creating a simple <Vg k="CT"/> task using the RESTful APIs.

### 1. Configure the request URL

Each RESTful API request follows a specific format. For example, the URL to create a transcoding task is:

```http
https://api.sd-rtn.com/v1/projects/{appId}/rtsc/cloud-transcoder/tasks
```

**Path parameters:**

* `https`: Secure protocol.
* `api.sd-rtn.com`: <Vg k="COMPANY"/>’s server domain.
* `v1/projects/{appId}/rtsc/cloud-transcoder/tasks`: Resource path.

  * `v1`: API version.
  * `{appId}`: Your <Vg k="COMPANY"/> app ID.
  * `rtsc/cloud-transcoder`: The product or service name.
  * `tasks`: The endpoint you’re calling.

<Admonition type="info" info="info">
Refer to the specific API documentation for the correct HTTP method (`GET`, `POST`, `PUT`, `PATCH`, `DELETE`) and resource path.
</Admonition>

### 2. Send the request

A typical API request includes:

* A URL
* Request headers
* (Optional) A request body

#### Request headers

| Header          | Description                                                                       |
| --------------- | --------------------------------------------------------------------------------- |
| `Accept`        | Expected response format, e.g., `application/json`.                               |
| `Authorization` | Required for authentication. See [RESTful Authentication](../reference/restful-authentication). |
| `Content-Type`  | Format of the request body, e.g., `application/json`.                             |


#### Request body

The request body contains the parameters specific to the API, such as input and output configurations.

#### Request example

```bash
curl --location --request POST 'https://api.sd-rtn.com/v1/projects/<your_app_id>/rtsc/cloud-transcoder/tasks' \
--header 'Accept: application/json' \
--header 'Authorization: <http_basic_auth>' \
--header 'Content-Type: application/json' \
--data '{
  "services": {
    "cloudTranscoder": {
      "serviceType": "cloudTranscoderV2",
      "config": {
        "transcoder": {
          "audioInputs": [
            {
              "rtc": {
                "rtcChannel": "test01",
                "rtcUid": 123,
                "rtcToken": "token_123"
              }
            },
            {
              "rtc": {
                "rtcChannel": "test01",
                "rtcUid": 456,
                "rtcToken": "token_456"
              }
            }
          ],
          "canvas": {
            "width": 1280,
            "height": 720,
            "color": 255
          },
          "videoInputs": [
            {
              "rtc": {
                "rtcChannel": "test01",
                "rtcUid": 123,
                "rtcToken": "token_123"
              },
              "placeholderImageUrl": "https://example.jpg",
              "region": {
                "x": 0,
                "y": 0,
                "width": 480,
                "height": 360,
                "zOrder": 2
              }
            },
            {
              "rtc": {
                "rtcChannel": "test01",
                "rtcUid": 456,
                "rtcToken": "token_456"
              },
              "placeholderImageUrl": "https://example.jpg",
              "region": {
                "x": 0,
                "y": 240,
                "width": 480,
                "height": 360,
                "zOrder": 2
              }
            }
          ],
          "outputs": [
            {
              "rtc": {
                "rtcChannel": "test02",
                "rtcUid": 1000,
                "rtcToken": "token_1000"
              },
              "audioOption": {
                "profileType": "AUDIO_PROFILE_MUSIC_STANDARD"
              },
              "videoOption": {
                "fps": 15,
                "codec": "H264",
                "width": 1280,
                "height": 720
              }
            }
          ]
        }
      }
    }
  }
}'
```

This example mixes audio and combines video from users `123` and `456` in channel `test01`, and outputs the stream to `test02` with user ID `1000`.

### 3. Handling the response

After sending a request, the server responds with:

* HTTP status code
* Response headers
* Response body

#### Response status codes

| Code  | Meaning | Action                                  |
| ----- | ------- | --------------------------------------- |
| 200   | Success | Proceed as normal.                      |
| !=200 | Error   | See [HTTP status codes](../reference/status-code). |

#### Response headers (Optional)

Common response headers include:

* `X-Request-ID`: Unique request identifier for tracking/debugging.

#### Example response

```json
{
  "taskId": "609f28f2644f1ae1ceb041b7047e3***",
  "createTs": 1661324613,
  "status": "STARTED",
  "services": {
    "cloudTranscoder": {
      "serviceType": "cloudTranscoderV2",
      "status": "serviceReady"
    }
  }
}
```

### 4. Troubleshoot

If a request fails:

1. Use browser or CLI developer tools to inspect the request and response.
2. Check the HTTP status code and refer to [HTTP status codes](../reference/status-code).
3. If the issue persists, contact [technical support](mailto:support@agora.io).
