This guide covers the configuration options available in the agent editor. The editor is organized into four sections: Live Audio Input, ASR, LLM, and TTS. Each section is accessible from the left panel or the flow diagram.

## Prerequisites

Follow the [quickstart](quickstart) to create your first <Vpd k="STUDIO" /> agent.

## Live audio input

The Live Audio Input section controls how your agent processes incoming audio and manages conversation turns. To open the configuration panel, select **Live Audio Input** from the left panel or the flow diagram.

![](/images/conversational-ai/studio/configure-audio-input.png)

### Turn detection

Turn detection controls when the agent considers you to have started or finished speaking. Enabling turn detection improves conversation flow by ensuring the agent listens and responds at the right moments.

- **Enable Turn Detection**: Toggle the switch to on to enable this feature.
  - **Type**: Currently fixed as `Agora VAD` (Voice Activity Detection).
  - **Threshold**: The minimum audio level required for the system to treat input as speech rather than background noise. Range: [0, 1]. Lower values make the agent more sensitive to low-volume speech; higher values help it ignore background noise.
  - **Silence Duration (ms)**: The duration of silence after which the agent considers you to have finished speaking and begins processing a response.
  - **Prefix Padding (ms)**: The amount of audio captured before and after detected speech to avoid cutting off the beginning or end of words.
  - **Interrupt Duration (ms)**: The duration in milliseconds that your voice must exceed the VAD threshold to trigger an interruption of the agent. Range: [80, 5000] ms. Default: 160 ms.
  - **Interruption Mode**: Controls how the agent handles being interrupted mid-response:
    - **Interrupt**: The agent stops speaking immediately when you start talking.
    - **Append**: Your speech does not interrupt the agent. The agent finishes its current response, then processes what you said.
    - **Ignore**: The agent continues speaking and discards your input.

For the full API reference, see [`turn_detection`](/conversational-ai/rest-api/agent/join#properties-turn-detection).

### Advanced features

- **AI Voice Activity Detection**: Enables AI-based end-of-speech detection using semantic understanding rather than silence alone. This results in more natural turn-taking by waiting for a complete thought rather than a pause. Currently available for English only. <Vg k="COMPANY" /> strongly recommends enabling this for optimal conversation quality.

- **Real-Time Messaging/Signaling**:  Enables the Agora Signaling (RTM) service for the agent. When enabled, the agent can deliver custom events and dynamic data to your client application in real time. 

- **Selective Attention Locking (SAL)**: Enables speaker-focused audio processing. The agent locks onto a single speaker and suppresses background voices and noise, which is useful in noisy environments. 

- **MCP Tool Invocation**: Enables the agent to call tools provided by MCP servers. MCP servers are configured in the LLM section and can also be managed centrally in [Integration](integration). See [MCP integration](integration#mcp) for details.

## ASR

The ASR component converts spoken audio into text for the LLM to process. To open the configuration panel, select **ASR** from the left panel or the flow diagram.

![ASR configuration](/images/conversational-ai/studio/configure-asr.png)

- **Vendor**: Select your ASR vendor.
- **Credential**: Select or create a credential for the API key provided by your ASR vendor. See [Add credential](integrations#credentials).
- **Language**: The BCP-47 language tag identifying the primary language used for agent interaction.

For vendor-specific parameters, select your vendor from [Supported ASR vendors](supported-vendors#asr-vendors).

## LLM 

The LLM processes user input and generates appropriate responses based on your configuration. To open the configuration panel, select **LLM** from the left panel or the flow diagram.

![LLM configuration](/images/conversational-ai/studio/configure-llm.png)

### Basic configuration

- **Vendor**: Select your LLM vendor.
- **Credential**: Select or create a credential for the API key provided by your LLM vendor. See [Add credential](integrations#credentials).
- **System Prompt**: The system prompt defines how your agent behaves. For example: `You are a helpful customer support assistant.`
- **Greeting Message**: The greeting is the first thing your agent says when a call starts. For example: `Hello! How can I help you today?`
- **Failure Message**: The message the agent speaks when the LLM fails to respond or returns an error. For example, `Please hold on a second.`
- **Max History**: The number of conversation turns the agent retains in context. Higher values give the agent more memory of the conversation but increase LLM token usage.

Configure additional parameters in the sidebar as needed. Available parameters vary by vendor. For parameter descriptions, select your vendor from [Supported LLM vendors](supported-vendors#llm-vendors).

### MCP

By configuring MCP servers, you can enable agents to call tools provided by external services to implement advanced functionality.

- **MCP Servers**: Specify one or more MCP servers to use.

  Click the **`+`** icon to select or create an MCP server integration. See [Add MCP server](integrations#mcp-servers).

### Knowledge base

A knowledge base consists of PDF or DOCX files. It provides detailed context to your agent.

- **Knowledge Base**: Specify one or more knowledge base integrations to use.

  Click the **`+`** icon to select or create a knowledge base integration. See [Add knowledge base](integrations#knowledge-bases).


## TTS 

The TTS component converts the LLM's text responses into natural-sounding speech. To open the configuration panel, select **TTS** from the left panel or the flow diagram.

![TTS configuration](/images/conversational-ai/studio/configure-tts.png)

- **Vendor**: Select your TTS vendor.
- **Credential**: Select or create a credential for the API key provided by your TTS vendor. See [Add credential](integrations#credentials).

Configure additional parameters in the sidebar as needed. Available parameters vary by vendor. For parameter descriptions, select your vendor from [Supported TTS vendors](supported-vendors#tts-vendors).
