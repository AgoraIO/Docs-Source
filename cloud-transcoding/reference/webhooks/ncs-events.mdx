## 110 `cloud_transcoder_started`

After successfully calling [`Create`](/cloud-transcoder/reference/rest-api/create-transcoding-task) to create a Cloud Transcoder, your server receives this event notification.

* **eventType**: `110` (`cloud_transcoder_started`)

**Sample payload**:

```json
{
  "taskId": "1234",
  "instanceId": "123",
  "sequence": 123,
  "sent": 1656573243385,
  "serviceScene": "rtsc/cloud-transcoder",
  "details": {
    "event": "cloud_transcoder_started",
    "state": "Transcoder success",
    "createTs": 1658112464000,
    "transcoder": {
      "inputs": [
        {
          "rtcChannel": "test01",
          "rtcUid": 123,
          "audio": true,
          "video": false
        },
        {
          "rtcChannel": "test01",
          "rtcUid": 456,
          "audio": true,
          "video": true
        }
      ],
      "outputs": [
        {
          "rtcChannel": "test01",
          "rtcUid": 1000,
          "audio": true,
          "video": true
        }
      ],
      "userConfigDetail": "{XXXX}"
    }
  }
}
```

**Payload parameters**:

* `event`: Callback event name (`"cloud_transcoder_started"`).
* `state`: Transcoder status (`"Transcoder success"`), indicating that transcoding has started.
* `createTs`: Creation timestamp (Unix time in seconds).
* `transcoder`:

  * `inputs[]`: Audio/video input configurations.

    * `rtcChannel`: Name of the RTC channel.
    * `rtcUid`: UID of the input stream.
    * `audio`: `true` if audio input is present.
    * `video`: `true` if video input is present.
  * `outputs[]`: Audio/video output configurations.

    * Same structure as `inputs`.
  * `userConfigDetail`: The `cloudTranscoder.config` value from the `Create` request.

## 113 `cloud_transcoder_updated`

Triggered after you successfully call [`Update`](/cloud-transcoder/reference/rest-api/update-transcoding-task) to update the Cloud Transcoder configuration.

* **eventType**: `113` (`cloud_transcoder_updated`)

**Sample payload**:

```json
{
  "taskId": "1234",
  "instanceId": "123",
  "sequence": 123,
  "sent": 1656573243385,
  "serviceScene": "rtsc/cloud-transcoder",
  "details": {
    "event": "cloud_transcoder_updated",
    "state": "running",
    "createTs": 1658112464000,
    "updateTs": 1658112464000,
    "transcoder": {
      "inputs": [
        {
          "rtcChannel": "test01",
          "rtcUid": 123,
          "audio": true,
          "video": false
        },
        {
          "rtcChannel": "test01",
          "rtcUid": 456,
          "audio": true,
          "video": true
        }
      ],
      "outputs": [
        {
          "rtcChannel": "test01",
          "rtcUid": 1000,
          "audio": true,
          "video": true
        }
      ],
      "userConfigDetail": "json_string"
    }
  }
}
```

**Payload parameters**:

* `event`: `"cloud_transcoder_updated"`
* `state`: Transcoder status. `"running"` means it's active and transcoding.
* `createTs`: Creation timestamp.
* `updateTs`: Last update timestamp.
* `userConfigDetail`: Updated config value (`cloudTranscoder.config`).
* `transcoder`: Same structure as in the `cloud_transcoder_started` event.

## 112 `cloud_transcoder_status`

Sent when the Cloud Transcoder's runtime state changes after it is created.

* **eventType**: `112` (`cloud_transcoder_status`)

**Sample payload**:

```json
{
  "taskId": "1234",
  "instanceId": "123",
  "sequence": 123,
  "sent": 1656573243385,
  "serviceScene": "rtsc/cloud-transcoder",
  "details": {
    "event": "cloud_transcoder_status",
    "state": "running",
    "createTs": 1658112464000,
    "updateTs": 1658112464000
  }
}
```

**Payload parameters**:

* `event`: `"cloud_transcoder_status"`
* `state`: Current transcoding status:

  * `"connecting"`: Connecting to Agora servers.
  * `"running"`: Transcoding is in progress.
  * `"failed"`: Transcoding has failed.
* `createTs`: Creation timestamp.
* `updateTs`: Last status update timestamp.

## 111 `cloud_transcoder_stopped`

Sent when the Cloud Transcoder is destroyed, either manually or due to an error.

* **eventType**: `111` (`cloud_transcoder_stopped`)

**Sample payload**:

```json
{
  "taskId": "1234",
  "instanceId": "123",
  "sequence": 123,
  "sent": 1656573243385,
  "serviceScene": "rtsc/cloud-transcoder",
  "details": {
    "event": "cloud_transcoder_stopped",
    "stopReason": "Stop request",
    "createTs": 1658112464000,
    "updateTs": 1658112464000
  }
}
```

**Payload parameters**:

* `event`: `"cloud_transcoder_stopped"`
* `stopReason`: Reason for stopping (e.g., client `Delete` call or an error).
* `createTs`: Creation timestamp.
* `updateTs`: Destruction timestamp.
