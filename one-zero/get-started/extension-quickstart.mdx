This page shows you how to create a Hello World extension for <Vg k="AI_FRAMEWORK" /> step by-step. You can choose to build the extension in Python, Go, or C++ according to your preference.

## Prerequisites

Before starting, ensure that you understand the basics of the <Vg k="AI_FRAMEWORK" />. Following the [Quickstart] guide to build and run the <Vg k="AI_VOICE_AGENT" /> will get you up to speed quickly.

## Set up

First things first, let’s start by composing the servers. 

1. **Compose up servers**

    Run the following command:

    ```bash
    docker compose up
    ```

    Once the command is entered, you should see output similar to this:


1. **Enter the docker container**

    To work within the isolated environment, run the following command:

    ```bash
    docker exec -it astra_agents_dev bash
    ```

### Implementation

1. **Create the extension**

    By running the following commands, an extension called hello_world will be created in Python, Go, or C++.

    <PlatformWrapper platform="python">
    ```bash
    cd agents/ten_packages/extension

    tman install extension default_extension_python \\
        --template-mode \\
        --template-data package_name=hello_world \\
        --template-data class_name_prefix=HelloWorld

    cd -
    ```
    </PlatformWrapper>

    <PlatformWrapper platform="go">

    ```bash
    cd agents/ten_packages/extension

    tman install extension default_extension_go \\
        --template-mode \\
        --template-data package_name=hello_world \\
        --template-data class_name_prefix=HelloWorld

    cd -
    ```
    </PlatformWrapper>

    <PlatformWrapper platform="cpp">
    ```bash
    cd agents/ten_packages/extension

    tman install extension default_extension_cpp \\
        --template-mode \\
        --template-data package_name=hello_world \\
        --template-data class_name_prefix=HelloWorld

    cd -
    ```
    </PlatformWrapper>

    After you run the command, you see a message similar to the following:

    ```bash
    ...
    Resolving packages...
    :-)  Installed successfully in xxx seconds
    ...
    ```

1. **Adding API to the extension**

    Navigate into the `hello_world` directory and open `manifest.json`. Add the API objects with `data_in` and `cmd_out`, which we will use shortly within the Graph Designer:

    ```json
    {
        "type": "extension",
        "name": "hello_world",
        "version": "0.4.1",
        "language": "python",
        "dependencies": [
            {
                "type": "system",
                "name": "rte_runtime_python",
                "version": "0.4.1"
            }
        ],
        "api": {
            "data_in": [
                {
                    "name": "text_data",
                    "property": {
                        "text": {
                            "type": "string"
                        },
                        "is_final": {
                            "type": "bool"
                        }
                    }
                }
            ],
            "cmd_out": [
                {
                    "name": "flush"
                }
            ]
        }
    }
    ```

1. **Build the extension**

    Switch back to the root of the project, and build the extension.

    ```bash
    cd -

    make build
    ```

1. **Restart the server**

    You don’t need to restart the server when you first build the agent. However, after making minor updates, if refreshing the page doesn’t apply the changes, you’ll need to restart the server in Docker to ensure the updates take effect.

    ![](/images/one-zero/restart-server.gif)

1. **Verify the extension**

    Open http://localhost:3001 in your browser. You should see hello_world in the left menu. Drag it to the canvas, and connect it to the text_data input and flash output.

    You see the green and red color indicting the possible routes of node connecting. 

    ![](/images/one-zero/graph-designer-hello.gif)

    Congratulations! You’ve successfully created your first hello_world extension, and it’s working seamlessly within the Graph Designer canvas.

1. **Check the network requests**

    Open Chrome DevTools, navigate to the Network panel, and monitor the requests. You should see the status codes returning as 200, indicating that the changes have been successfully processed.
