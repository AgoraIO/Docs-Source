## `BasicAuthCredential`

| API    | Description      |
|--------|------------------|
| [`BasicAuthCredential`](#basicauthcredential) | Creates a `BasicAuthCredential` instance. |

Initializes a new `BasicAuthCredential` instance using your <Vg k="COMPANY"/> Customer ID and Customer Key.

```java

```

#### Parameters

| Parameter  | Data type | Description                                                                               |

| `username` | `String`  | The Customer ID from <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="COMPANY"/> Console</Link>.  |

## `ConvoAIClient`

| API                            | Description                                                              |
|--------------------------------|--------------------------------------------------------------------------|
| [`create`](#create)            | Creates a conversational AI engine REST Client with the specified configuration. |
| [`join`](#join)                | Creates an agent instance and joins the specified RTC channel.           |
| [`leave`](#leave)              | Stops the specified agent instance and leaves the RTC channel.           |
| [`list`](#list)                | Retrieves a list of agents that match the specified criteria.            |
| [`query`](#query)              | Queries the current state of the specified agent instance.               |
| [`update`](#update)            | Adjusts the parameters of the agent at runtime.                          |

### `create`

Creates a `ConvoAIClient` instance using the provided configuration.

```java
public static synchronized ConvoAIClient create(ConvoAIConfig convoAIConfig)
```

#### Parameters

| Parameter       | Data type       | Description                                                                            |
| --------------- | --------------- | -------------------------------------------------------------------------------------- |
| `convoAIConfig` | `ConvoAIConfig` | Configuration for the client. See [`ConvoAIConfig`](#convoaiconfig) for details. |

#### Return value

A new `ConvoAIClient` instance.

### `join`

Creates an agent instance and joins the specified RTC channel.

```java
public abstract Mono<JoinConvoAIRes> join(JoinConvoAIReq request);
```

#### Parameters

| Parameter | Data type        | Description                                                                                         |
| --------- | ---------------- | --------------------------------------------------------------------------------------------------- |
| `request` | `JoinConvoAIReq` | Parameters required to join the channel. See [`JoinConvoAIReq`](#joinconvoaireq) for details. |

#### Return value

A `JoinConvoAIRes` instance containing the join response. See [`JoinConvoAIRes`](#joinconvoaires).

### `leave`

Stops the specified agent instance and leaves the RTC channel.

```java
public abstract Mono<Void> leave(String agentId);
```

<Admonition type="info">
Before using this method, ensure that you have called the `join` API to obtain the agent ID.
</Admonition>

#### Parameters

| Parameter | Data type | Description              |
| --------- | --------- | ------------------------ |
| `agentId` | `String`  | ID of the agent to stop. |

### `list`

Retrieves a list of agents that match the specified criteria.

```java
public abstract Mono<ListConvoAIRes> list(ListConvoAIReq request);
```

#### Return value

A `ListConvoAIRes` instance containing the matching agents. See [`ListConvoAIRes`](#listconvoaires).

### `query`

Returns the current state of the specified agent instance.

```java
public abstract Mono<QueryConvoAIRes> query(String agentId);
```

<Admonition type="info">
Before using this method, ensure that you have called the `join` API to obtain the agent ID.
</Admonition>

#### Parameters

| Parameter | Data type | Description               |
| --------- | --------- | ------------------------- |
| `agentId` | `String`  | ID of the agent to query. |

#### Return value

A `QueryConvoAIRes` instance containing the current state. See [`QueryConvoAIRes`](#queryconvoaires).

### `update`

Updates the agent's parameters at runtime.

```java
public abstract Mono<UpdateConvoAIRes> update(String agentId, UpdateConvoAIReq request);
```

<Admonition type="info">
Before using this method, ensure that you have called the `join` API to obtain the agent ID.
</Admonition>

#### Parameters

| Parameter | Data type          | Description                                                               |
| --------- | ------------------ | ------------------------------------------------------------------------- |
| `agentId` | `String`           | ID of the agent to update.                                                |
| `request` | `UpdateConvoAIReq` | Configuration updates. See [`UpdateConvoAIReq`](#updateconvoaireq). |

#### Return value

An `UpdateConvoAIRes` instance with the update results. See [`UpdateConvoAIRes`](#updateconvoaires).

### Properties

Configure the properties of the agent joining the RTC channel.

```java 
public static class Properties { 
    @JsonProperty("token") 
    private String token; 
    @JsonProperty("channel") 
    private String channel; 
    @JsonProperty("agent_rtc_uid") 
    private String agentRtcUId; 
    @JsonProperty("remote_rtc_uids") 
    private List<String> remoteRtcUIds; 
    @JsonProperty("enable_string_uid") 
    private Boolean enableStringUId; 
    @JsonProperty("idle_timeout") 
    private Integer idleTimeout; 
    @JsonProperty("agent_rtm_uid") 
    private String agentRtmUId; 
    @JsonProperty("advanced_features") 
    private AdvancedFeatures advancedFeatures; 
    @JsonProperty("llm") 
    private LLMPayload llmPayload; 
    @JsonProperty("tts") 
    private TTSPayload ttsPayload; 
    @JsonProperty("vad") 
    private VADPayload vadPayload; 
    @JsonProperty("asr") 
    private ASRPayload asrPayload; 
} 
``` 

#### Parameters 

| Parameter            | Type              | Description |
|---------------------|-------------------|-------------|
| `token`             | `String`          | Dynamic key (**Token**) for joining the RTC channel, used for authentication (optional). If <strong>App Certificates</strong> are enabled, this must be passed. |
| `channel`           | `String`          | The name of the RTC channel that the agent joins (required). |
| `agentRtcUId`       | `String`          | The user ID of the agent in the RTC channel (required). "0" means random assignment, but the token must be modified accordingly. |
| `remoteRtcUIds`     | `List<String>`    | The list of user IDs that the agent subscribes to in the RTC channel. Only subscribed users can interact with the agent (required). `"*"` subscribes to all users. |
| `enableStringUId`   | `Boolean`         | Whether to enable string UID (optional). `true`: Enable, `false`: Disable (default). |
| `idleTimeout`       | `Integer`         | Max idle time (in seconds) of the RTC channel (optional). After all `remoteRtcUIds` users leave, this time counts as idle. `0` means manual stop only. |
| `agentRtmUId`       | `String`          | The user ID of the agent in the RTM channel. Used only when `advancedFeatures.enable_rtm` is `true`. |
| `advancedFeatures`  | `AdvancedFeatures`| Advanced configuration (optional), see [`AdvancedFeatures`](#advancedfeatures). |
| `llmPayload`        | `LLMPayload`      | Custom language model (LLM) config (required), see [`LLMPayload`](#llmpayload). |
| `ttsPayload`        | `TTSPayload`      | Text-to-speech (TTS) config (optional), see [`TTSPayload`](#ttspayload). |
| `vadPayload`        | `VADPayload`      | Voice Activity Detection (VAD) config (optional), see [`VADPayload`](#vadpayload). |
| `asrPayload`        | `ASRPayload`      | Automatic Speech Recognition (ASR) config (optional), see [`ASRPayload`](#asrpayload). |

### `LLMPayload`

Defines the custom language model (LLM) configuration for the agent to join the RTC channel.

```java 
public static class LLMPayload { 
    @JsonProperty("url") 
    private String url; 
    @JsonProperty("api_key") 
    private String apiKey; 
    @JsonProperty("system_messages") 
    private List<Map<String, Object>> systemMessages; 
    @JsonProperty("params") 
    private HashMap<String, Object> params; 
    @JsonProperty("max_history") 
    private Integer maxHistory; 
    @JsonProperty("greeting_message") 
    private String greetingMessage; 
    @JsonProperty("input_modalities") 
    private List<String> inputModalities; 
    @JsonProperty("output_modalities") 
    private List<String> outputModalities; 
    @JsonProperty("failure_message") 
    private String failureMessage; 
} 
``` 

#### Parameters

| Parameter         | Type                        | Description |
|------------------|-----------------------------|-------------|
| `url`            | `String`                    | LLM callback URL (required). Must be compatible with the OpenAI protocol. |
| `apiKey`         | `String`                    | LLM authentication API key (required). Default is empty, API key must be enabled in production environments. |
| `systemMessages` | `List<Map<String, Object>>` | A set of predefined messages appended at the beginning of each LLM call, used to control the LLM output (optional). Can be character settings, prompts and answer examples, must be compatible with the OpenAI protocol. |
| `params`         | `HashMap<String, Object>`   | Additional information transmitted in the LLM message body, such as the model used, maximum token limit, etc. (optional). Different LLM providers support different configurations, see their respective LLM documentation for details. |
| `maxHistory`     | `Integer`                   | Number of short-term memory entries cached in the LLM (optional). Default is 10. Pass 0 to not cache short-term memory. Agents and subscribers will log entries separately. |
| `greetingMessage`| `String`                    | Agent greeting message (optional). If filled in, the agent will automatically send a greeting message to the first subscriber who joins the channel when there are no users in the `remote_rtc_uids` list. |
| `inputModalities`| `List<String>`              | Input modes for the LLM (optional).<br />- `["text"]`: Text only (default).<br />- `["text", "image"]`: Text and image, requires the selected LLM to support visual mode input. |
| `outputModalities`| `List<String>`             | Output modes for the LLM (optional).<br />- `["text"]`: Text only (default), the output text will be converted to speech by the TTS module and then published to the RTC channel.<br />- `["audio"]`: Audio only. The audio will be published directly to the RTC channel.<br />- `["text", "audio"]`: Text and audio. You can write your own logic to process the LLM output as needed. |
| `failureMessage` | `String`                    | Agent's failure message (optional). If filled in, it will be returned by the TTS module when the LLM call fails. |

### `VADPayload`

Defines the voice activity detection (VAD) configuration for the agent to join the RTC channel. 

```java 
public static class VADPayload { 
    @JsonProperty("interrupt_duration_ms") 
    private Integer interruptDurationMs; 
    @JsonProperty("prefix_padding_ms") 
    private Integer prefixPaddingMs; 
    @JsonProperty("silence_duration_ms") 
    private Integer silenceDurationMs; 
    @JsonProperty("threshold") 
    private Float threshold; 
} 
``` 

#### Parameters

| Field                 | Type     | Description |
|----------------------|----------|-------------|
| `interruptDurationMs` | `Integer` | Voice duration threshold (milliseconds), range [120, 1200] (optional). The minimum duration of a continuous voice signal to avoid false interruptions. |
| `prefixPaddingMs`     | `Integer` | Prefix padding threshold (milliseconds), range [0, 5000] (optional). The minimum speech duration required to start a new speech segment to avoid triggering voice activity detection due to very short sounds. |
| `silenceDurationMs`   | `Integer` | Silence duration threshold (milliseconds), range [0, 2000] (optional). Minimum duration of silence at the end of speech, to ensure that short pauses do not end speech segments prematurely. |
| `threshold`           | `Float`   | Speech recognition sensitivity, range (0.0, 1.0) (optional). Determines the level of sound in the audio signal that is considered "speech activity". Lower values make it easier for the agent to detect speech, while higher values may ignore faint sounds. |

### `ListConvoAIReq`

Defines the filter condition type used to query the list of conversational agents.

public class ListConvoAIReq { 
    private final String channel;

    private final String cursor ;

    private final Long toTime; 
    private final Integer state; 
}

#### Parameters

| Field      | Type     | Description |
|------------|----------|-------------|
| `channel`  | `String` | Set the channel name. |
| `limit`    | `Integer` | Set the number of items. Limit the number of items returned per page, the default value is 20. |
| `cursor`   | `String` | Set the paging cursor. The paging cursor is the agent ID of the starting position of the paging. |
| `fromTime` | `Long`   | Set the end timestamp. The end timestamp (seconds), the default value is the current time. |
| `toTime`   | `Long`   | Set the start timestamp. Start timestamp (seconds), the default value is 1 day ago. |
| `state`    | `Integer` | Query the agent state, the following states are supported:<br/>- `IDLE (0)`: The agent is idle.<br/>- `STARTING (1)`: The agent is starting.<br/>- `RUNNING (2)`: The agent is running.<br/>- `STOPPING (3)`: The agent is stopping.<br/>- `STOPPED (4)`: The agent has stopped.<br/>- `RECOVERING (5)`: The agent is recovering.<br/>- `FAILED (6)`: The agent has failed. |
