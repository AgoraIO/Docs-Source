This page guides you through creating and testing your first AI voice agent using <Vpd k="STUDIO" />. In less than 10 minutes, you will have a working agent that you can speak to in the browser. No telephony setup is required.

## Prerequisites

Make sure you have the following:

- An active Agora account and project
- [Enabled Conversational AI](/conversational-ai/get-started/manage-agora-account) for your Agora project

## Create an agent

To create your first agent:

1. Log into [Agora Console](https://console.agora.io) and open [<Vpd k="STUDIO" />](https://console.agora.io/studio).
    ![](/images/conversational-ai/studio/create-agent.png)
1. Select **Agents** from the sidebar, then click **Create Agent**.
1. Select an Agora project from the dropdown. The agent uses this project's credentials to connect to Agora, and usage is billed to this project.
    ![](/images/conversational-ai/studio/create-agent-template.png)
1. Under **Choose a template**, select the **Appointment Reminder** template.
1. Enter a name for your agent.
1. Click **Create Agent**.

Studio opens the agent editor with a system prompt, greeting message, and pre-configured model settings.

![](/images/conversational-ai/studio/agent-editor-prompt.png)

## Configure your agent

The agent editor is organized into tabs. For the quickstart, you only need to review the **Prompt** tab and configure the **Models** tab.

### Review the prompt

The agent editor opens with the following pre-configured prompts from the Appointment Reminder template:

1. **System prompt**: It defines how your agent behaves. In this sample case, it is confirming, rescheduling, or cancelling appointments.
1. **Greeting message**: It is the first thing your agent says when a session starts.
1. **Failure message**: It is used when the LLM encounters an error or fails to respond.

<Admonition type="info">
The prompt uses dynamic variables such as `{{appointment_date}}` and `{{appointment_time}}`. These are replaced with real values when the agent makes a call. You can leave everything as-is for the quickstart.
</Admonition>

### Select models

Models determine which ASR, LLM, and TTS services your agent uses to process speech, generate responses, and synthesize voice output.

1. Click the **Models** tab.
    ![](/images/conversational-ai/studio/agent-editor-models.png)
1. Under **Automatic Speech Recognition (ASR)**, select a vendor-model from the dropdown.
1. Under **Large Language Model (LLM)**, select a vendor-model from the dropdown.
1. Under **Text-to-Speech (TTS)**, select a vendor-model and a **Voice** from the dropdowns.

The dropdowns list vendor-model combinations that support Agora Managed Key. When you use a managed key, Agora provides the API credentials for your selected vendor and bills you directly for usage. You do not need to obtain an API key directly from the vendor. See [Pricing](/conversational-ai/overview/pricing). To use your own API key or use a vendor that does not support a managed key, see [Customize your agent](build/customize-agent).

## Test the agent

1. In the right panel, click **Start Call** to start a test call.
1. Allow microphone access when prompted.
1. Speak to the agent as a caller would. Try confirming, rescheduling, and cancelling an appointment to test different conversation paths.

For guidance on what to test and how to troubleshoot issues, see [Test your agent](build/test-agent).
