OBS is a popular open source software for live streaming scenarios. It supports various host features and multiple streaming protocols. Starting with version 30.0.0, OBS officially supports the WebRTC-HTTP Ingestion Protocol (WHIP), for establishing WebRTC media connections. <Vg k="COMPANY"/> offers an experimental feature for WHIP protocol integration. 

This guide explains how to stream media from OBS to a <Vg k="VSDK" /> channel using the WHIP protocol.

## Set up OBS Studio

This section explains how to configure OBS Studio to stream media to a channel.

If you don't have OBS Studio or the installed version is too old, download and install the latest version from the [official website](https://obsproject.com/).

### Configure video parameters

Follow these steps to configure OBS Studio video parameters:  

1. Open OBS and navigate to **Settings**:  
   - macOS: **OBS > Preferences**  
   - Windows: **File > Settings**
2. Switch to the **Output** tab and set the **Output Mode** to **Advanced**.
3. Configure the following parameters:
   - **Video Encoder**: `x264`
   - **Keyframe Interval**: `2s`
   - **CPU Usage Preset**: `veryfast`
   - **Profile**: `baseline` (recommended) or use the default settings.
   - **Tune**: `zerolatency` (recommended) or use the default value.
4. Click **OK** to save the changes.

You only need to configure these parameters once. 

![OBS Video Parameter Configuration](/images/video-sdk/obs-video-output-settings.png)

### WHIP access configuration  

Follow these steps to configure WHIP access settings:

1. Open OBS and navigate to **Settings**:  
   - macOS: **OBS > Preferences**  
   - Windows: **File > Settings**
2. Switch to the **Stream** tab and set **Service** to **WHIP**.  
   OBS prompts you to switch the encoder to **FFmpeg Opus**. Click **Yes** to confirm.
3. Configure the following:
   - **Server**: A valid [WHIP server address](#whip-server-address)
   - **Bearer Token**: A valid [WHIP authentication token](#whip-tokens).
4. Click **OK** to save the changes.

![OBS WHIP Parameter Configuration](/images/video-sdk/obs-whip-configuration.png)

<Admonition type="info" info="info">

- **Bearer tokens** are time-sensitive. When a token expires, you must generate a new one and update in OBS. To generate bearer tokens, see [WHIP authentication](#whip-authentication)

- To enhance the user experience and eliminate manual configuration, use WebSocket services for remote WHIP parameter configuration. Refer to [Remotely configure WHIP](#remotely-configure-whip-optional) for details.
</Admonition>

## Implement OBS streaming

This section explains how to stream media to a channel using OBS Studio.

### Start streaming  

After completing the video and WHIP access configurations, streaming users can initiate streaming by clicking **Start Streaming** in OBS. This action pushes the WHIP protocol stream to the channel.

![OBS Starts Pushing WHIP](/images/video-sdk/start-obs-streaming.png)

### Remotely configure WHIP (Optional)

OBS supports WHIP streaming and WebSocket services, enabling developers to remotely configure WHIP access parameters. This feature reduces manual setup and improves the live-streaming experience. To use WebSocket to configure and control OBS remotely:

1. The streaming user completes the video configuration and leaves OBS open.
1. The application connects to the local OBS instance using WebSocket.
1. The application remotely configures WHIP parameters and manages the starting and stopping of the live broadcast.

#### Enable WebSocket in OBS  

To enable WebSocket services, follow these steps (required only once):

1. Open the **Tools** menu and select **WebSocket Server Settings**.
2. In the **WebSocket Server Settings** interface:
   - Enable the WebSocket server by checking **Enable WebSocket server**.
   - Disable authentication by unchecking **Enable Authentication**.
3. Click **OK** to save the changes.

![OBS Opens WebSocket](/images/video-sdk/obs-web-socket-config.png)

1. Configure WHIP access parameters:

    Before proceeding, ensure the following:

    1. The streaming user has switched OBS to the **Live** tab and set the service to **WHIP**.
    2. Streaming is stopped before modifying WHIP parameters.

    ```javascript
    let cmdId = 1;

    /**
    * Updates WHIP server settings
    * @param {Object} config - WHIP configuration
    * @param {string} config.address - Agora WHIP server address
    * @param {string} config.appid - Agora App ID
    * @param {string} config.streamid - Channel name
    * @param {string} config.token - Authentication token
    */
    function updateServerSettings(config) {
        socket.send(
            JSON.stringify({
            op: 6,
            d: {
                requestId: `${cmdId++}`,
                requestType: "SetStreamServiceSettings",
                requestData: {
                streamServiceType: "whip_custom",
                streamServiceSettings: {
                    server: `https://${config.address}/push/${config.appid}/${config.streamid}`,
                    service: "WHIP",
                    bearer_token: config.token,
                },
                },
            },
            })
        );
    }
    ```

1. Start streaming:

    ```javascript
    function startStream() {
        socket.send(
            JSON.stringify({
            op: 6,
            d: {
                requestId: `${cmdId++}`,
                requestType: "StartStream",
                requestData: {},
            },
            })
        );
    }

    function handleWebSocketMessage(event) {
        let data = JSON.parse(event.data);
        if (data.op === 5 && data.d.eventType === "StreamStateChanged") {
            console.log("Streaming Status:", data.d.eventData.outputState);
            if (data.d.eventData.outputState === "BS_WEBSOCKET_OUTPUT_STARTED") {
            console.log("Streaming Started");
            }
        }
    }
    ```

1. To check the streaming status, query the streaming duration and activity:

    ```javascript
    function getStatus() {
        socket.send(
            JSON.stringify({
            op: 6,
            d: {
                requestId: `${cmdId++}`,
                requestType: "GetStreamStatus",
                requestData: {},
            },
            })
        );
        }

        function handleWebSocketMessage(event) {
        let data = JSON.parse(event.data);
        if (data.op === 7 && data.d.requestType === "GetStreamStatus") {
            if (data.d.requestStatus.code === 100 && data.d.responseData.outputActive) {
            console.log("Streaming Duration:", data.d.responseData.totalStreamTime);
            }
        }
    }
    ```

1. You can stop streaming at any time:

    ```javascript
    function stopStream() {
        socket.send(
            JSON.stringify({
            op: 6,
            d: {
                requestId: `${cmdId++}`,
                requestType: "StopStream",
                requestData: {},
            },
            })
        );
        }

        function handleWebSocketMessage(event) {
        let data = JSON.parse(event.data);
        if (data.op === 5 && data.d.eventType === "StreamStateChanged") {
            console.log("Streaming Status:", data.d.eventData.outputState);
            if (data.d.eventData.outputState === "BS_WEBSOCKET_OUTPUT_STOPPED") {
            console.log("Streaming Stopped");
            }
        }
    }
    ```

### WHIP authentication

To enable WHIP streaming, the app developer must generate the server address and bearer token and provide them to the streaming user. 

#### WHIP server address   

The WHIP server address must be formatted as:  

```
https://{server_address}/push/{appid}/{streamid}
```

Where:  

- `server_address`: <Vg k="COMPANY"/> provides multiple access line addresses to ensure connectivity for users in different regions. Use one of the following:  
  - `ap-webrtc-whip.ap.sd-rtn.com`  
  - `ap-webrtc-whip.agora.io`  
- `appid`: The project ID generated when you create a project in the <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link>.  
- `streamid`: The name of the channel to which the stream will be pushed.  

#### WHIP tokens  

The Bearer token for WHIP streaming must be in JWT format and consist of three parts: Header, Payload, and Signature. Follow these guidelines to generate the token:  

1. **Header**: The header must include:  
   - `alg`: Specifies the signature algorithm. For WHIP, use `HS256`.  
   - `typ`: Specifies the token type. Use `JWT`.  

   Example:  
   ```json
   {
     "alg": "HS256",
     "typ": "JWT"
   }
   ```  

2. **Payload**: The payload must include:  
   - `version`: Token version. Use `"1.0"`.  
   - `appId`: The project ID from <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link>.  
   - `streamId`: The name of the channel to which the stream will be pushed.  
   - `exp`: The token's expiration timestamp (in seconds), which must occur before the user starts streaming.  
   - `action`: Required value: `"push"`.  

   Example:  
   ```json
   {
     "version": "1.0",
     "appId": "<your appid>",
     "streamId": "<your stream id>",
     "exp": 1733388002,
     "action": "push"
   }
   ```  

3. **Signature**: 

    Use the App Certificate for your project from <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link> to generate the signature.
