Implementing a Content Security Policy (CSP) enhances the security of your web application by allowing only trusted content to be executed. CSP is a security standard designed to prevent a wide range of attacks, including Cross-Site Scripting (XSS) and other code injection attacks. It enables you to control the resources a user agent can load for a given page. By specifying a set of directives in the CSP, you restrict the accessible sources, thereby reducing the attack surface for malicious content.

## Implement a Content Security Policy 

To implement a Content Security Policy, configure your web server to add a `Content-Security-Policy` header to the HTTP responses. Refer to the following sample policy tailored for use with Agora Signaling. It is designed to provide a secure environment while allowing necessary communications with <Vg k="SIG" /> services.

```
default-src 'none';
script-src 'self' 'wasm-unsafe-eval';
style-src 'self';
child-src 'self';
worker-src 'self' blob:;
connect-src 'self'
https://*.agora.io wss://.agora.io:
https://*.sd-rtn.com wss://.sd-rtn.com:
https://*.edge.agora.io wss://.edge.agora.io:
https://*.edge.sd-rtn.com wss://.edge.sd-rtn.com:
https://*.ap.sd-rtn.com wss://.ap.sd-rtn.com:
https://*.statscollector.sd-rtn.com wss://.statscollector.sd-rtn.com:
https://*.logservice.sd-rtn.com wss://.logservice.sd-rtn.com:;
font-src 'self';
frame-src 'self' ;
img-src 'self';
manifest-src 'none';
media-src 'self';
object-src 'self';
form-action 'self';
frame-ancestors 'self';
```
