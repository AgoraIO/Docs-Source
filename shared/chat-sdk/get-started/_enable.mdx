Before using <Vg k="CHAT" />, you need to enable and configure it through <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link>.

## Prerequisites

To enable <Vg k="CHAT" />, make sure that you have the following:

- A valid [<Vg k="COMPANY" /> account](/agora-chat/reference/manage-agora-account#create-an-agora-account).
- An [<Vg k="COMPANY" /> project](/agora-chat/reference/manage-agora-account#create-an-agora-project) that uses  **App ID** and **Token** for authentication.
- A <Vg k="CHAT" /> pricing plan. For details on how to subscribe, see <Link to="../overview/pricing#subscribe-to-the-pricing-plan">Subscribe to the pricing plan</Link>.


## Enable <Vg k="CHAT" />

1. Log in to the <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link>.

1. In the left navigation bar, click **Projects**. Under **My Projects**, select the project that you want to enable <Vpd k="NAME" /> for and click the **Edit** ✎ button.

    ![configure-edit=project](/images/chat/configure-edit-project.png)

1. In the **ALL FEATURES** section, click **Chat**. 

    ![configure-select-data-center](/images/chat/configure-select-data-center.png)

1. To enable <Vg k="CHAT" />, select a data center closest to where most of your end users are located.

1. If this is your first <Vg k="CHAT" /> project, select and subscribe to a <Vg k="CHAT" /> package. In the left navigation bar, click **Subscriptions**. Choose **Agora Chat** and select a subscription package.

    ![](/images/chat/configure-subscribe-package.png)


1. To enable the advanced features based on your business requirements, Select **ALL FEATURES > Chat > Features > Overview**:

    ![](/images/chat/configure-features-overview.png)

    For details about these advanced features, see the following:
    - <Link to="../develop/setup-webhooks">Message Callback</Link>
    - <Link to="../client-api/messages/send-receive-messages#recall-a-message">Message Recall</Link>
    - <Link to="../client-api/threading/thread-management">Message Thread</Link>
    - <Link to="../client-api/reaction">Reaction</Link>
    - <Link to="../develop/offline-push">Offline Message Push (Advanced)</Link>
    - <Link to="../client-api/presence">Presence</Link>
    - <Link to="../client-api/messages/translate-messages">Translation</Link>
    - <Link to="../develop/content-moderation">Moderation</Link>

## Get Chat project information

<Vg k="CONSOLE" /> assigns the following information to each project that enables <Vg k="CHAT" />:

- **Data Center**: <Vg k="COMPANY" /> provides several data centers for the service in different regions, including Singapore, Frankfurt (Germany), and Virginia (USA). After the plan is changed, the data center remains unchanged.
- **AppKey**: The unique identifier that <Vg k="CHAT" /> assigns to each app. The **AppKey** is of the form `${OrgName}#{AppName}`.
    <Admonition type="info">
    The **AppKey** is not the same as your project App ID. You can get your **AppKey** from the Chat configuration section of <Vg k="CONSOLE" />.
    </Admonition>
- **OrgName**: The unique identifier that <Vg k="CHAT" /> assigns to each enterprise (organization).
- **AppName**: The name that <Vg k="CHAT" /> assigns to each app. Each app under the same enterprise (organization) must have a unique App Name.
- **API request url**: The domain of the WebSocket and RESTful API request that <Vg k="COMPANY" /> assigns to each project.

Follow these steps to get the project information:

1. Find your <Vg k="CHAT" />-enabled project on the <Link to="{{Global.AGORA_CONSOLE_URL}}/project-management">Project Management</Link> page on <Vg k="CONSOLE" /> and click **Config**.
2. On the project edit page, find **<Vg k="CHAT" />** and click **Config**.
3. On the <Vg k="CHAT" /> config page, get the values of **Data Center**, **AppKey**, **OrgName**, **AppName**, **WebSocket Address**, and **REST API**.

## Manage users and generate tokens

For development purposes, <Vg k="COMPANY"/> enables you to manage users and generate <Vg k="CHAT" /> user authentication tokens using <Vg k="CONSOLE"/>. In a production environment, you use the [RESTful API](../restful-api/user-system-registration) to manage users and a token server to [generate user authentication tokens](../develop/authentication).

This section shows you how to register <Vpd k="NAME" /> users and generate temporary tokens using <Vg k="CONSOLE"/>.

### Register a user

To register a user, do the following:

1. On the **Projects** page, select the project that you want to use and click the **Edit** ✎ button.

    ![configure-edit=project](/images/chat/configure-edit-project.png)

2. On the **Edit Project** page, select **Chat > Operation Management > User** and click **Create User**.

    ![configure-edit=project](/images/chat/configure-create-user.png)

4. In the **Create User** dialog box, fill in the **User ID** and click **Save** to create a user.

	![configure-edit=project](/images/chat/configure-create-user-form.png)

### Generate an app token

In the **Basic Information** section of **Chat**, click **Generate** next to **Chat App Temp Token** to generate a token with app privileges.

    ![token_generate_app_token](/images/chat/configure-app-token.png)

### Generate a user token

To ensure communication security, <Vg k="COMPANY" /> recommends using tokens to authenticate users who log in to <Vg k="CHAT" />.

For testing purposes, <Vg k="CONSOLE" /> supports generating temporary tokens for <Vg k="CHAT" />. To generate a user token, do the following:

In the **Basic Information** section of **Chat**, enter the [user ID](#userid) in the **Chat User Temp Token** box and click **Generate** to generate a token with user privileges.

	![token_generate_user_token](/images/chat/configure-user-token.png)

## Change the <Vg k="CHAT" /> plan

1. Log in to the <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link>.

2. In the left navigation bar, click **Subscriptions**, then select **Agora Chat**.

3. On the **Agora Chat Plans** page, you can change your <Vg k="CHAT" /> plan by clicking **Upgrade** next to the desired plan.

   ![Agora Chat Plans](/images/chat/configure-subscribe-package.png)

   <Admonition type="info" title="Note">
   - The plan change takes effect immediately.
   - <Vg k="COMPANY" /> doesn't recommend plan downgrading, as it could impact the capacity of your applications and availability of certain features.
   </Admonition>

## Disable <Vg k="CHAT" />

1. Log in to the <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link>.

2. In the left navigation bar, click **Project Management** and click **Config** for the project that you want to use.

3. In the **Features** section of the **Edit Project** page, click **Enable/Configure** next to **Chat**.

4. On the **Application Information** page, disable <Vg k="CHAT" /> in the **Chat Service Status** section.
   
   **Note:** When you disable <Vg k="CHAT" />, the Chat-related data is still stored in the <Vg k="CHAT" /> data center, and you are still charged for the plan activation fee.

   ![img](https://web-cdn.agora.io/docs-files/1665387330975)


## Unsubscribe <Vg k="CHAT" />

Before unsubscribing <Vg k="CHAT" />, disable all projects that have <Vg k="CHAT" /> enabled.

1. Log in to the <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link>.

2. In the left navigation bar, click **Package**.

3. On the **Subscribe** tab, scroll down to the bottom and click **Unsubscribe**.

   ![img](https://web-cdn.agora.io/docs-files/1665389814916)

    **Note:**

    When you unsubscribe <Vg k="CHAT" />:
    - All <Vg k="CHAT" />-related data is purged.
    - You are still billed with a pro-rated monthly fee and the usage that occurred during that month.
