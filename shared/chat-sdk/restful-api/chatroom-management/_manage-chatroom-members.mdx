This page shows how to manage <Vg k="CHAT_ROOM_LOWER"/> members by calling the Chat RESTful APIs, including adding, retrieving, and deleting <Vg k="CHAT_ROOM_LOWER"/> members and revoking administrative privileges of <Vg k="CHAT_ROOM_LOWER"/> administrators.
Before calling the following methods, ensure that you understand the call frequency limit of the Chat RESTful APIs described in <Link to="/agora-chat/reference/limitations#call-limit-of-server-sides">Limitations</Link>.

## Description of <Vg k="CHAT_ROOM_LOWER"/> member roles

| Role | Description | Privilege |
| :----------- | :--------------------------------------------------- | :----------------------------------------------------------------------------- |
| Regular member | Chat room members without administrative privileges. | Regular <Vg k="CHAT_ROOM_LOWER"/> members can modify their own <Vg k="CHAT_ROOM_LOWER"/> profiles. |
| Chat room admin | Chat room admins are authorized by the <Vg k="CHAT_ROOM_LOWER"/> owner and have <Vg k="CHAT_ROOM_LOWER"/> management privileges. | The admins can manage regular members of the <Vg k="CHAT_ROOM_LOWER"/>.  A maximum of 99 admins can be added. |
| Chat room owner | The creator of the <Vg k="CHAT_ROOM_LOWER"/>. Chat room owners have the highest privileges. | The <Vg k="CHAT_ROOM_LOWER"/> owner can add <Vg k="CHAT_ROOM_LOWER"/> admins, disband the <Vg k="CHAT_ROOM_LOWER"/>, modify <Vg k="CHAT_ROOM_LOWER"/> information, and manage regular members of the <Vg k="CHAT_ROOM_LOWER"/>. |

## Adding a <Vg k="CHAT_ROOM_LOWER"/> member

Adds the specified user to the <Vg k="CHAT_ROOM_LOWER"/>.

For each App Key, the call frequency limit of this method is 100 per second.

### HTTP request

```html
POST https://{host}/{org_name}/{app_name}/chatrooms/{chatroomid}/users/{username}
```

#### Path parameter

For the parameters and detailed descriptions, see [Common parameters ](#param).

> If the specified user does not exist in the app or is already a member of the <Vg k="CHAT_ROOM_LOWER"/>, the request fails and the error code `400` is returned.

#### Request header

| Parameter | Type | Description | Required |
| :-------------- | :----- | :--------------------- | :------- |
| `Content-Type` | String | `application/json` | Yes |
| `Accept` | String | `application/json` | Yes |
| `Authorization` | String | The authentication token of the user or administrator, in the format of `Bearer ${token}`, where `Bearer` is a fixed character, followed by an English space, and then the obtained token value. | Yes |

### HTTP response

#### Response body

If the returned HTTP status code is `200`, the request succeeds, and the response body contains the following fields:

| Field | Type | Description |
| :------- | :----- | :------------------------------------------------------ |
| `result` | Bool | The addition result:<li>`true`: Success</li><li>`false`: Failure</li> |
| `action` | String | The operation that is performed. `add_member` means to add a member to the <Vg k="CHAT_ROOM_LOWER"/>. |
| `id` | String | The <Vg k="CHAT_ROOM_LOWER"/> ID. This is the unique identifier assigned to the <Vg k="CHAT_ROOM_LOWER"/> by the Chat service. |
| `user` | String | The array of usernames of the existing <Vg k="CHAT_ROOM_LOWER"/> members. |

If the returned HTTP status code is not `200`, the request fails. You can refer to [Status codes](#code) for possible reasons.

### Example

#### Request example

```json
# Replace <YourAppToken> with the app token generated in your server.
curl -X POST -H 'Content-Type: application/json' -H 'Accept: application/json' -H 'Authorization: Bearer <YourAppToken>' 'http://XXXX/XXXX/XXXX/chatrooms/XXXX/users/XXXX'
```

#### Response example

```json
{
    "action": "post",
    "application": "8be024f0-XXXX-XXXX-b697-5d598d5f8402",
    "uri": "http://XXXX/XXXX/XXXX/chatrooms/XXXX/users/XXXX",
    "entities": [],
    "data": {
        "result": true,
        "action": "add_member",
        "id": "XXXX",
        "user": "user1"
    },
    "timestamp": 1542554038353,
    "duration": 0,
    "organization": "XXXX",
    "applicationName": "XXXX"
}
```
