## Prerequisites

To use Convo AI Studio, you need:

- An active Agora account 
- An Agora project with Conversational AI enabled
- API credentials for your chosen providers (LLM, ASR, TTS)
- For telephony features: A configured SIP domain and phone numbers


## Create a pipeline

### Access ConvoAI Studio

Log in to <Vg k="CONSOLE" />. From the left sidebar, select **Convo AI Studio**.

### Create a new pipeline

Select **Agent Pipelines** from the left sidebar. 
Click the **New Pipeline** button.
Select **Conversational AI Agent**
Enter any name for your pipeline
Select a project from the dropdown or create a new one
After creating your pipeline, you'll be taken to the pipeline creator page where you can configure your agent.


Click on the ASR component in the pipeline editor
![](/images/conversational-ai/studio/pipeline-asr-config.png)

Configure ASR (Speech Recognition)
  In the Configure ASR panel, click on the Credential dropdown
  Click on '+ Create New Credential' button
  ![](/images/conversational-ai/studio/add-credential.png)

  Enter a credential name and paste the API key below
  Select the credential you created in the ASR configuration panel

Configure LLM (Language Model)

  Click on the LLM component in the pipeline

  Select 'OpenAI' as the vendor

  Click on the Credential dropdown and select '+ Create New Credential'

  Make sure the credential you created is selected in the LLM configuration panel

  Copy and paste a System Prompt such as "You are a helpful chatbot" into the System Messages field in the LLM configuration panel

  Copy and paste a greeting message such as "Hello, How are you today?" into the Greeting field in the LLM configuration panel

Configure TTS (Text-to-Speech)
  Click on the TTS component in the pipeline
  In the Configure TTS panel, click on the Credential dropdown
  Click on '+ Create New Credential' button
  Select 'ElevenLabs' from the vendor dropdown
  Enter a credential name and paste the ElevenLabs API key below
  Make sure the credential you created is selected in the TTS configuration panel

Save pipeline configuration
