In under five minutes, you will have a working voice agent running locally. The guide covers installing the Agora CLI, cloning a starter project in your preferred language, and loading your Agora credentials.

<Admonition type="info">
The starter projects use Agora-managed presets, so no vendor API keys are required to complete this quickstart. If you prefer to switch to your own vendor accounts, see the documentation for your chosen [ASR](/conversational-ai/models/asr/overview), [LLM](/conversational-ai/models/llm/overview), and [TTS](/conversational-ai/models/tts/overview) providers.
</Admonition>

## Prerequisites

- An active [Agora account](https://console.agora.io). No Agora console configuration is required. The Agora CLI handles project creation, feature enablement, and credential retrieval.
- One of the following runtimes:
  - **Node 20 LTS or later** for the TypeScript/Next.js starter
  - **Python 3.8 or later** and **Bun** for the Python starter
- `git` and a terminal.
- Roughly five minutes.

## Set up your environment

This section walks you through installing the Agora CLI and signing in to your Agora account.

### Install the Agora CLI

Agora CLI is the recommended way to bootstrap a new Agora project. Use it to create projects, enable features, write credentials to `.env` files, and run a diagnostic.

```bash
npm install -g agoraio-cli
```

Verify the installation and sign in with your Agora account:

```bash
agora --version
agora login
agora whoami
```

`agora login` opens a browser for SSO. On a headless machine, use `agora login --device` to complete the OAuth flow using a six-digit code.

<details>
<summary>Migrating from `agora-cli-preview`?</summary>

`agora-cli-preview` is deprecated. Uninstall it before installing the GA release:

```bash
npm uninstall -g agora-cli-preview
npm install -g agoraio-cli
```

Command names are unchanged in the GA release; configuration is re-read from `~/.agora-cli/` on first run.
</details>

### Create or select an Agora project

If you do not yet have a Conversational AI project, create one now:

```bash
agora project create my-agent-demo --feature rtc --feature convoai
```

If you already have a project, select it as the active project so subsequent commands use it:

```bash
agora project list
agora project use <project-id>
```

The CLI now has access to your App ID, App Certificate, and Conversational AI feature status. No console configuration is required.

## Clone and run a starter

Select a starter in your preferred language, load your Agora credentials, and run it locally.

### Clone the repository

<Tabs groupId="language">
<TabItem value="js" label="TypeScript (Next.js)">

The Next.js project is a starting point for a production app. It includes a React UI, three API routes (`generate-agora-token`, `invite-agent`, `stop-conversation`), and a hook-based transcript view.

Clone the repository and install dependencies:

```bash
git clone https://github.com/AgoraIO-Conversational-AI/agent-quickstart-nextjs.git
cd agent-quickstart-nextjs
pnpm install
```

If you do not have `pnpm`, install it first:

```bash
npm install -g pnpm
```

- `server-python/`: The FastAPI backend
- `web-client/`: The Next.js web client

Bun workspaces connect the two folders, with Python running the agent server.

Clone the repository and install dependencies:

```bash
git clone https://github.com/AgoraIO-Conversational-AI/agent-quickstart-python.git
```

If you do not have Bun, install it first:


### Load your Agora credentials

Each starter includes a `.env` template. Run the following commands to load your credentials and create the env file:

<Tabs groupId="language">
<TabItem value="js" label="TypeScript (Next.js)">

The Next.js starter expects `NEXT_PUBLIC_AGORA_APP_ID` and `NEXT_AGORA_APP_CERTIFICATE` in `.env.local`.

```bash
# Export your Agora credentials to the shell from the active CLI project
eval "$(agora project env --shell)"

# Copy the env template and overwrite with your credentials
cp env.local.example .env.local
cat > .env.local <<EOF
NEXT_PUBLIC_AGORA_APP_ID=$AGORA_APP_ID
NEXT_AGORA_APP_CERTIFICATE=$AGORA_APP_CERTIFICATE
EOF
```

The remaining variables in `env.local.example`, such as `NEXT_PUBLIC_AGENT_UID`, `NEXT_LLM_*`, `NEXT_DEEPGRAM_API_KEY`, and `NEXT_ELEVENLABS_API_KEY`, are optional. Leave them blank to use Agora-managed presets.

</TabItem>
<TabItem value="python" label="Python">

The Python starter expects `APP_ID` and `APP_CERTIFICATE` in `server-python/.env.local`.

```bash
# Export your Agora credentials to the shell from the active CLI project
eval "$(agora project env --shell)"

# Copy the env template and overwrite with your credentials
cp server-python/.env.example server-python/.env.local
cat > server-python/.env.local <<EOF
APP_ID=$AGORA_APP_ID
APP_CERTIFICATE=$AGORA_APP_CERTIFICATE
PORT=8000
EOF
```

</TabItem>

<Admonition type="info">
The CLI command `agora project env write` writes credentials directly to a file in one step, but it uses Agora's default variable names `AGORA_APP_ID` and `AGORA_APP_CERTIFICATE`. The starters use framework-specific variable names. The `eval` and heredoc commands in this section are more portable for these cases. If you control the starter repo, you can rename the variables and use `agora project env write` directly.
</Admonition>

Start the development server and open the app in your browser.


Open `http://localhost:3000`, click **Start conversation**, and speak. You should:

1. Hear a greeting within two seconds of clicking **Start**.
2. See live partial and final transcripts appear as you speak.
3. Hear the agent respond in a natural voice.

1. Hear a greeting within two seconds of clicking **Start**.

### Test and validate

Confirm the following before moving on. A healthy run passes all three checks within the specified time budgets.

| Check | How to verify | Time budget |
|---|---|---|
| Agent joined the channel | The starter UI shows a connected status and the agent emits a greeting. | < 2 s |
| First audio arrives | You hear the greeting after clicking **Start**. | < 2 s |
| Transcripts stream | Partial transcripts update as you speak. Final transcripts appear after each utterance. | < 500 ms |

If any check fails, run:

```bash
agora project doctor
```

The command checks credential validity, feature enablement, and network reachability to Agora's edge servers. It catches most first-run issues without requiring code changes.

### Stop the agent

Close the browser tab or press `Ctrl+C` in the terminal to stop the dev server. The starter calls the `stop-conversation` endpoint on unload, which terminates the agent cleanly. The <Vpd k="NAME" /> also applies an idle timeout after 30 seconds of silence.

You can stop an orphaned agent manually using its ID:

```bash
curl -X POST http://localhost:3000/api/stop-conversation \
  -H 'Content-Type: application/json' \
  -d '{"agentId":"<agent-id-from-logs>"}'
```

## Troubleshooting

To troubleshoot first-run failures, run `agora project doctor`. The command diagnoses issues caused by invalid credentials, missing feature enablement, or network reachability problems.

| Symptom | Likely cause | Fix |
|---------|--------------|-----|
| `agora: command not found` after `npm install -g agoraio-cli` | Global npm bin directory is not on `$PATH`. | Run `npm prefix -g` to find the global prefix, then add `<prefix>/bin` to your shell configuration file. |
| `401 unauthorized` in the browser console | The env file was not populated with CLI values. | Re-run the `eval` and heredoc commands in the credentials step and verify the file contents with `cat`. |
| Greeting plays but no transcripts appear | `enable_rtm` is off, or the browser blocked the RTM WebSocket. | The starters set `enable_rtm: true` by default. Check the browser console and your network policy for blocked `wss://` connections. |
| `pnpm: command not found` | `pnpm` is not installed. | Run `npm install -g pnpm`. The TypeScript starter requires pnpm. |
| `bun: command not found` | Bun is not installed. | Run `curl -fsSL https://bun.sh/install \| bash`. The Python starter requires Bun. |
| Agent greets itself in a loop | Microphone and speaker on the same device without echo cancellation. | Use headphones, or set `parameters.enable_aec=true` in the starter's agent configuration. |
| `agora login` fails on a headless machine | Browser SSO cannot open. | Use `agora login --device` to authenticate using a six-digit code instead. |

If the issue is not listed here, open an issue on the starter repo and attach the output of `agora project doctor`. This output contains the diagnostics the maintainers need to resolve your issue.
