## Get started with <Vg k="COMPANY"/>

To join a <Vpd k="NAME" /> session, you need an <Vg k="COMPANY"/> account and project. This section shows you how to set up an <Vg k="COMPANY"/> account, create an <Vg k="COMPANY"/> project and get the required credentials from <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link>.

<SignInSignUp />

### Create an <Vg k="COMPANY"/> project

To create an <Vg k="COMPANY"/> project, do the following:

1.  In <Link to="{{Global.AGORA_CONSOLE_URL}}"><Vg k="CONSOLE" /></Link>, open the <Link to="{{Global.AGORA_CONSOLE_URL}}/project-management">Projects</Link> page.

1.  Click **Create New**.

1.  Follow the on-screen instructions to enter a project name and use case, and check **Secured mode: APP ID + Token (Recommended)** as the authentication mechanism.

    ![configure_project](/images/signaling/create_new_project.png)

1.  Click **Submit**. You see the new project on the **Projects** page.

### Get the App ID

<ProductWrapper notAllowed="conversational-ai">
The App ID uniquely identifies your <Vg k="COMPANY" /> project.
</ProductWrapper>
<ProductWrapper product="conversational-ai">
The App ID uniquely identifies your <Vg k="COMPANY" /> project. It is used to route Conversational AI Engine API calls to the associated project.

To obtain your App ID, find your project on the <Link to="{{Global.AGORA_CONSOLE_URL}}/project-management">Projects</Link> page in  <Vg k="CONSOLE" />, and click the copy icon in the **App ID** column.

## Security and authentication

Use the following credentials from your <Vg k="COMPANY"/> account to implement security and authentication features in your apps.

### Get the App Certificate

<ProductWrapper notAllowed="conversational-ai">
The App Certificate is used to generate secure authentication tokens on your app server.
</ProductWrapper>
<ProductWrapper product="conversational-ai">
The App Certificate is used on your backend server to generate RTC tokens that allow Conversational AI agents to authenticate and join RTC channels securely.
</ProductWrapper>

To get an App Certificate:

1.  On the <Link to="{{Global.AGORA_CONSOLE_URL}}/project-management">Projects</Link> page, click the pencil icon to edit the project you want to use.

    ![Console project management page](/images/common/console-project-management-page.png)

1.  Click the copy icon under **Primary Certificate**.

    ![Console primary certificate](/images/common/console-primary-certificate.png)

### Generate temporary tokens

<ProductWrapper notAllowed="conversational-ai">
Tokens authenticate users when they join RTC channels. To ensure communication security, best practice is to use tokens to authenticate the users who log in from your <Vpl k="CLIENT" />. Generate authentication tokens on your backend server using your App ID and App Certificate. Never generate tokens on the client side.
</ProductWrapper>

<ProductWrapper product="conversational-ai">
Tokens authenticate users and agents when they join RTC channels. Generate authentication tokens on your backend server using your App ID and App Certificate. Never generate tokens on the client side.
</ProductWrapper>

To generate a temporary RTC token for development and testing:

1. On the <Link to="{{Global.AGORA_CONSOLE_URL}}/project-management">Projects</Link> page, click the pencil icon next to your project.

1. On the **Security** panel, click **Generate Temp Token**, enter a channel name in the pop-up box and click **Generate**. Copy the generated RTC token for use in your <Vpd k="NAME" /> projects.

<ProductWrapper notAllowed="conversational-ai">
To generate a token for other <Vg k="COMPANY"/> products:

1. In your browser, navigate to the [<Vg k="COMPANY" /> token builder](https://agora-token-generator-demo.vercel.app/).

1. Choose the <Vg k="COMPANY" /> product your user wants to log in to. Fill in **App ID** and **App Certificate** with the
details of your project in <Vg k="CONSOLE" />.

1. Customize the token for each user. The required fields are visible in the <Vg k="COMPANY" /> token builder.

    <ProductWrapper product="signaling">
    In <Vg k="MESS" />, each authentication token you create is specific for a user ID in your <Vpl k="CLIENT" />. You create a token for each user in the channel. When you call `login` using <Vg k="MESS_SDK" />, ensure that the UID is the same as you used to create the token.
    </ProductWrapper>

1. Click **Generate Token**.

    The token appears in Token Builder.

1. Copy the token and use it in your <Vpl k="CLIENT" />.

</ProductWrapper>

For production environments, see [Deploy a token server](/video-calling/token-authentication/deploy-token-server) to learn how to integrate token generation in your backend server.
