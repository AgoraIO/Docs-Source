For the communication between end users behind enterprise firewalls, <Vg k="COMPANY"/> Chat allows all communications to always use TCP/TLS 443 with TLS headers. <Vg k="COMPANY"/> Chat supports the IP allowlist function. If you have a strict security policy for end users and allow only certain absolute domain names or specified IP addresses/port ranges to communicate through TCP/TLS 443, you can add the allowed IP addresses or domain names to the IP allowlist. To ensure smooth communication via <Vg k="COMPANY"/> Chat, you must add the IP addresses and ports of data centers of <Vg k="COMPANY"/> Chat to the firewall allowlist.

### Add IP addresses and ports of data centers of <Vg k="COMPANY"/> Chat to the firewall allowlist

1. Prepare the development environment. See [SDK quickstart](../get-started/get-started-sdk) for your platform.

1. Add the IP addresses and ports of <Vg k="COMPANY"/> Chat access points to the firewall allowlist.

1. Test message sending and receiving in the development environment.
