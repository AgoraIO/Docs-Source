After starting a recording, call this endpoint to query its current status.

<Admonition type="info">
This endpoint is only valid within an active recording session. If the recording was not started successfully or has already ended, the request returns `404`.
</Admonition>

## Request

### Path parameters

<PathParameter name="appid" type="string" required={true}>
The App ID of your project.

- For web page recording mode, enter the App ID for which the cloud recording service is enabled.
- For individual and composite recording modes, use the same App ID as the channel to be recorded. Ensure that the cloud recording service has been enabled for this App ID.
</PathParameter>

<PathParameter name="resourceid" type="string" required={true}>
The resource ID obtained from the [`acquire`](./acquire) endpoint.
</PathParameter>

<PathParameter name="sid" type="string" required={true}>
The recording ID obtained from the [`start`](./start) endpoint.
</PathParameter>

<PathParameter name="mode" type="string" required={true}>
The recording mode:

- `individual`: [Individual recording mode](../develop/individual-mode).
- `mix`: [Composite recording mode](../develop/composite-mode).
- `web`: [Web page recording mode](../develop/webpage-mode).
</PathParameter>

## Response

### Response body

- If the returned status code is `200`, the request was successful.

  <ParameterList title="OK">
    <Parameter name="cname" type="string">
    The name of the channel being recorded.
    </Parameter>

    <Parameter name="uid" type="string">
    The UID used by the cloud recording service in the RTC channel.
    </Parameter>

    <Parameter name="resourceId" type="string">
    The resource ID used by cloud recording.
    </Parameter>

    <Parameter name="sid" type="string">
    The recording ID, identifying the current recording session.
    </Parameter>

    <Parameter name="serverResponse" type="object">
    The recording status. The fields returned vary depending on the recording mode and whether screenshot capture is enabled.

    <Tabs groupId="server-response-scenarios">
      <TabItem value="web" label="Web page recording">

      Fields returned in web page recording mode.

         <ParameterList title="Fields returned in web page recording mode">
        <Parameter name="status" type="number">
        Current status of the cloud service:

        - `0`: Not started.
        - `1`: Initialization complete.
        - `2`: Components starting.
        - `3`: Some components ready.
        - `4`: All components ready.
        - `5`: In progress.
        - `6`: Stop requested.
        - `7`: All components stopped.
        - `8`: Exited.
        - `20`: Exited abnormally.
        </Parameter>

        <Parameter name="extensionServiceState" type="array[object]">
        State of each extension service.

          <Parameter name="serviceName" type="string">
          Extension service name:

          - `"web_recorder_service"`: Web page recording service.
          - `"rtmp_publish_service"`: CDN push service.
          </Parameter>

          <Parameter name="payload" type="object">
          Service-specific state.

            <Parameter name="state" type="string">
            Upload status:

            - `"init"`: Initializing.
            - `"inProgress"`: Running.
            - `"exit"`: Stopped.
            </Parameter>

            <Parameter name="onhold" type="boolean">
            Whether recording is paused. `true` if paused, `false` if running. Only returned by `web_recorder_service`.
            </Parameter>

            <Parameter name="fileList" type="array[object]">
            Files generated during recording. Only returned by `web_recorder_service`.

              <Parameter name="filename" type="string">
              Name of the M3U8 or MP4 file.
              </Parameter>

              <Parameter name="sliceStartTime" type="number">
              Recording start time of the file, Unix timestamp in seconds.
              </Parameter>
            </Parameter>

            <Parameter name="outputs" type="array[object]">
            CDN push output status. Only returned by `rtmp_publish_service`.

              <Parameter name="rtmpUrl" type="string">
              The CDN address the stream is being pushed to.
              </Parameter>

              <Parameter name="status" type="string">
              Push status:

              - `"connecting"`: Connecting to the CDN server.
              - `"publishing"`: Push in progress.
              - `"onhold"`: Push paused.
              - `"disconnected"`: Failed to connect to the CDN server.
              </Parameter>
            </Parameter>
          </Parameter>
        </Parameter>
        </ParameterList>

      </TabItem>
      <TabItem value="screenshot" label="Individual recording with screenshot">

      <ParameterList title="Fields returned in individual recording mode when screenshot capture is enabled.">  

        <Parameter name="sliceStartTime" type="number">
        Recording start time, Unix timestamp in milliseconds.
        </Parameter>
      </ParameterList>  

      </TabItem>
      <TabItem value="other" label="Other scenarios">

      <ParameterList title="Fields returned in all other cases, including individual recording without screenshot capture and composite recording.">
        <Parameter name="status" type="number">
        Current status of the cloud service:

        - `0`: Not started.
        - `1`: Initialization complete.
        - `2`: Components starting.
        - `3`: Some components ready.
        - `4`: All components ready.
        - `5`: In progress.
        - `6`: Stop requested.
        - `7`: All components stopped.
        - `8`: Exited.
        - `20`: Exited abnormally.
        </Parameter>

        <Parameter name="fileListMode" type="string">
        Data format of `fileList`:

        - `"string"`: In composite recording mode when `avFileType` is `["hls"]`.
        - `"json"`: When `avFileType` is `["hls","mp4"]` in individual or composite recording mode.
        </Parameter>

        <Parameter name="fileList" type="string or array[object]">
        When `fileListMode` is `"string"`, this is the filename of the M3U8 file. When `fileListMode` is `"json"`, this is an array of objects with the following fields:

          <Parameter name="fileName" type="string">
          Name of the M3U8 or MP4 file.
          </Parameter>

          <Parameter name="trackType" type="string">
          File type:

          - `"audio"`: Audio only.
          - `"video"`: Video only.
          - `"audio_and_video"`: Audio and video.
          </Parameter>

          <Parameter name="uid" type="string">
          The UID whose stream is recorded. In composite recording mode, this is `"0"`.
          </Parameter>

          <Parameter name="mixedAllUser" type="boolean">
          `true` if all users are recorded in a single file; `false` if each user is recorded separately.
          </Parameter>

          <Parameter name="isPlayable" type="boolean">
          `true` if the file can be played online; `false` if it cannot.
          </Parameter>

          <Parameter name="sliceStartTime" type="number">
          Recording start time of the file, Unix timestamp in seconds.
          </Parameter>
        </Parameter>

        <Parameter name="sliceStartTime" type="number">
        Recording start time, Unix timestamp in milliseconds.
        </Parameter>
      </ParameterList>  

      </TabItem>
    </Tabs>

    </Parameter>
  </ParameterList>

- If the returned status code is not `200`, the request failed. See [Response status codes](../reference/common-errors#status-codes) for troubleshooting.
