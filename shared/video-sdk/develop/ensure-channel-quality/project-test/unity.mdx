5. **Choose this sample in the reference app**

    From the main screen of the <Vpl k="CLIENT" />, choose **<Vpd k="NAME" />** from the dropdown and then select **Call quality best practice**.

1. **Initialization best practice**

    When the <Vpl k="CLIENT" /> initializes the <Vg k="ENGINE" />, it does the following:

    * Sets the log file location, size, and logging level according to your preference.
    * Enables the dual-stream mode.
    * Sets the audio profile.
    * Sets the video profile.
    * Starts a network probe test.
    <ProductWrapper product={["interactive-live-streaming", "broadcast-streaming"]}>
    * Sets the audio latency level.
    </ProductWrapper>
    
    You see the result of the network probe test displayed in the network status icon.

1. **Run the audio/video device test**

    1. Press **Start Device Test**. You see the local camera feed in a window. 
    
    1. Speak into the device microphone. You hear the recorded audio after a short delay. 
    
        This test confirms that the user's hardware is working properly.

    1. Press **Stop Device Test** to end the test.

1. **Join a channel**

    <ProductWrapper product="{['interactive-live-streaming', 'broadcast-streaming']}">
    1. Click **Join** to start a session.
    2. Select a role using the toggles.
        - When you join as a **Host**, the local video is published and played in the <Vpl k="CLIENT"/>.
        - When you join as **Audience**, the remote stream is subscribed and played.
    </ProductWrapper>

    <ProductWrapper product="video-calling">
    Click **Join** to start a call. Now, you can see yourself on the device screen and talk to the remote user using your <Vpl k="CLIENT"/>.
    </ProductWrapper>

1. **Monitor network health**

    You see the network status indicator updated periodically based on the result of the `onNetworkQuality` callback.

1. **View connection statistics**

    After joining a channel, you receive messages in the debug console informing you of some selected call statistics, including: 

    * The number of users in the channel
    * Packet loss rate
    * Remote video state changes 

1. **Test the dual steam functionality**

    Click **High Video Quality**. The <Vpl k="CLIENT" /> switches the video quality to high. Click the same button to switch back to the low quality stream.
