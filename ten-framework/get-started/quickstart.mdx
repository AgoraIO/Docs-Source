The <Vg k="AI_VOICE_AGENT" />, built using the <Vg k="AI_FRAMEWORK" />, integrates <Vg k="COMPANY" />'s real-time voice interaction capabilities with its real-time speech-to-text service. The transcribed text is then used as input to a Large Language Model (LLM). The LLM's text response is subsequently sent to a text-to-speech cloud service and played back to the user as audio. This integration allows users to interact with AI in a natural, conversational manner.

This page uses the <Vg k="AI_VOICE_AGENT" /> as an example to introduce the capabilities of the <Vg k="AI_FRAMEWORK" />. It shows you how to set up the framework and build and run an app agent locally. 

## Understand the tech

The <Vg k="AI_VOICE_AGENT" /> integrates the following <Vg k="AI_FRAMEWORK" /> extensions:

- **Agora RTC**: Voice capture and transmission to Agora.
- **Agora RTT Text Converter**: Speech-to-text conversion.
- **Interrupt detector**: Detect and manage user interruptions for a natural conversational flow.
- **OpenAI ChatGPT**: LLM for AI response.
- **Azure TTS**: Text-to-speech service.

## Prerequisites

Before starting this project, ensure that you have:

- Hardware:
    - A CPU with 2 or more cores
    - At least 4 GB of RAM

- Software:
    - [Git](https://git-scm.com/downloads)
    - [Docker and Docker Compose](https://docs.docker.com/compose/)
    - [Node.js (LTS)](https://nodejs.org/en) v18 or above

- The following API keys:

    - [Agora App ID and App Certificate](/video-calling/get-started/manage-agora-account#get-the-app-id)
    - Azure [speech-to-text](https://azure.microsoft.com/en-us/products/ai-services/speech-to-text) (STT) and [text-to-speech](https://azure.microsoft.com/en-us/products/ai-services/text-to-speech) (TTS) keys
    - [OpenAI](https://openai.com/index/openai-api/) API key

<Admonition>
If you are developing on Apple Silicon, see [Docker setting on Apple Silicon](#docker-setting-on-apple-silicon).
</Admonition>

## Set up your project

To set up your project, take the following steps:

### Clone the repository

Navigate to the `<project-folder>` where you want to clone the repository and execute the following command:

```bash
git clone https://github.com/rte-design/astra.ai
```

### Create config files

Use the following commands to create config files from the examples provided in the repository:

```bash
# Create the .env file from .env.example 
cp ./.env.example ./.env

# Create the property.json file from property.json.example
cp ./agents/property.json.example ./agents/property.json
```

### Set up API keys 

Open the `.env` file in `<project-folder>/ASTRA.ai` and update the values for the following API keys:

```bash
# Agora App ID and App Certificate
AGORA_APP_ID=your-agora-app-id
AGORA_APP_CERTIFICATE=your-agora-app-certificate

# Azure STT key and region 
AZURE_STT_KEY=your-azure-speech-to-text-key
AZURE_STT_REGION=your-azure-speech-to-text-region

# Azure TTS key and region 
AZURE_TTS_KEY=your-azure-text-to-speech-key
AZURE_TTS_REGION=your-azure-text-to-speech-region

# OpenAI API key
OPENAI_API_KEY=your-openai-api-key
```

## Build and run the agent

Take the following steps:

1. **Compose Docker containers**

    In `<project-folder>/ASTRA.ai`, execute the following command to compose Docker containers:

    ```bash
    docker compose up
    ```

1. **Build your agent**

    To build your project using the Docker container’s interactive shell, execute the following commands:

    ```bash
    docker exec -it astra_agents_dev bash
    make build
    ```

    <Admonition type="info">
    If you see the `/usr/bin/env: 'bash\r': No such file or directory` message, open the file `astra.ai/agents/scripts/install_deps_and_build.sh` in your editor and convert the end of line sequence in the file from `CRLF` to `LF`.
    </Admonition>

1. **Start the server**

    Execute the following command

    ```bash
    make run-server
    ```

## Test your agent

1. Open https://localhost:3000 in your browser to load your <Vg k="AI_VOICE_AGENT" />.

1. Type in a **Username** and press **Join**.

1. When prompted for microphone and camera access, press **Allow** to grant the necessary permissions.

1. Press **Connect** to join a channel. You hear an audio message and see a welcome text on the screen.

1.  Use your microphone to interact with the <Vg k="AI_VOICE_AGENT" />. You see and hear the responses from the AI agent.
